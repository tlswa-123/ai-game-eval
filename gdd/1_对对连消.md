## 0.5 Coding Agent 导读

### A. 核心机制摘要

```
🎯 一句话核心：在16×10网格中，点击两个相同图案方块，若两者间存在≤2个折弯的无障碍路径则消除，消除后上方方块下落填补

⚡ 与其他消除类的关键差异：
1. 不是相邻交换三消，而是**任意距离点击配对消除**——两个相同图案间必须存在无障碍路径
2. 不是无限转弯，而是**≤2次直角折弯**——路径最多2个拐角（3段线段）
3. 不是消除后从顶部生成新元素，而是**仅现有方块下落填补**——无新随机方块注入
4. 不是拖拽画线，而是**依次点击两个方块**——点击选中→点击配对
5. 不是颜色匹配，而是**图案ID严格相等**——图案必须完全一致
6. 路径**不可走网格外围虚拟边界**——本项目路径仅在网格内部检测（与经典连连看的边界外可走不同）
```

### B. 前置否定清单

```
🚫 以下游戏行为绝对不要实现：
❌ 拖拽交换相邻方块——玩家点击选中两个方块，不是滑动交换
❌ 斜向/对角线连接——路径仅支持水平和垂直直线段，折角必须为直角
❌ 3个及以上同时消除——每次只消除2个相同图案的方块（成对消除）
❌ 顶部生成新方块填补——仅已有方块重力下落，无新元素从外部注入
❌ 超过2个折弯的路径——路径最多2次直角折弯（3段线段）
❌ 路径穿过未消除的方块或障碍——路径只能经过空格
❌ 真实物理引擎——下落和消除均用Tween缓动动画
❌ 限时倒计时机制——本项目无时间压力（代码证据确认）
⚠️ 包含以上任何一项，说明核心设计理解错误。
```

### C. 反例描述

```
🚨 错误实现特征：

1. ❌ 错误现象：交换相邻方块形成三连消除
   ✅ 正确现象：点击任意位置的两个相同图案方块，不需相邻，路径连通即消除
   📌 验证方法：选两个隔5格但路径可通的同图案方块→点击→应能消除

2. ❌ 错误现象：路径穿过有方块的格子
   ✅ 正确现象：路径只能经过已消除的空格位置
   📌 验证方法：两个同图案方块之间隔着其他方块→直线路径应判定为不连通

3. ❌ 错误现象：路径超过2个折弯（如3个拐角）也能消除
   ✅ 正确现象：路径最多2个直角折弯，超过则判定不连通
   📌 验证方法：构造一对需要3次转弯才能连通的同图案→应不能消除

4. ❌ 错误现象：消除后新方块从顶部随机生成填入
   ✅ 正确现象：仅上方已有方块下落填补空位，不生成任何新方块
   📌 验证方法：消除底部方块→上方方块下落→顶部出现空位→空位保持为空

5. ❌ 错误现象：消除后不检查是否还有可消除对
   ✅ 正确现象：每次消除+下落后，全盘检测可消除对数，0则触发死局处理
   📌 验证方法：构造只剩不可连通的对→应触发刷新道具提示或自动重排

6. ❌ 错误现象：连通路径画直线而非折线
   ✅ 正确现象：路径必须展示实际走过的折线段（1-3段）
   📌 验证方法：消除时→应看到沿实际路径绘制的折线连接

7. ❌ 错误现象：关卡生成时某图案出现奇数个
   ✅ 正确现象：每种图案必须成对（偶数个），保证理论上可全部消除
   📌 验证方法：统计初始网格每种图案数量→全部为偶数

8. ❌ 错误现象：冰块方块一次消除就消失
   ✅ 正确现象：冰块有3层（-6→-5→-4→正常），每次参与消除邻居时层级减1
   📌 验证方法：点击冰块旁的配对→冰块层级降低但不消失→需多次才能变为普通方块
```

## 3. 系统模块

### 3.1 路径连通消除系统（核心）

#### 模块目标

玩家通过观察+推理，找到可连通消除的同图案方块对并点击消除，逐步清空16×10网格。

#### 核心规则

**行为链 Step1: 选中方块**

| 维度 | 描述 |
|------|------|
| **触发** | 点击网格中有方块的格子 |
| **前置检查** | ✅ 该格有方块且未消除且未被障碍完全锁定 / ❌ 空格→不响应 / ⚠️ 动画中→锁定输入 |
| **执行** | 无选中→设为选中(绿色高亮+scale1.1) / 已有选中→进入配对判定 / 点击同一个→取消选中 |
| **后果** | 选中态等待第二次点击 |
| **终止条件** | 选中完成 |
| **显式否定** | ❌ 不能选空格；❌ 不能拖拽 |
| **时序约束** | 选中响应<16ms，高亮动画100ms |

**行为链 Step2: 图案匹配 + 路径连通判定**

| 维度 | 描述 |
|------|------|
| **触发** | 已有selectA + 点击另一个方块selectB |
| **前置检查** | ✅ selectA.id == selectB.id（图案ID相同） / ❌ 图案不同→直接切换选中到selectB |
| **执行** | 调用路径检测算法：搜索从A到B的路径，仅走空格，折弯次数≤2 |
| **后果-连通** | 画出金色折线(300ms) → 消除两方块(缩至0, 200ms) → 粒子碎裂(300ms) → 空位产生 |
| **后果-不连通** | 显示红色×标 + 方块抖动(300ms) → 切换选中到selectB |
| **终止条件** | 消除完成或切换完成 |
| **显式否定** | ❌ 路径不能穿过方块/障碍；❌ 折弯不能超过2次；❌ 不能斜向移动 |
| **时序约束** | 路径搜索<10ms → 折线动画300ms → 消除200ms → 粒子300ms |

**路径碰撞检测规则（🔒核心——Coding Agent 必读）**

| 规则 | 描述 | 🔒强制 |
|------|------|--------|
| **方向约束** | 路径仅支持水平/垂直正交移动，❌不支持对角线 | 🔒 |
| **最大折弯数** | 路径最多2个拐角（3段线段） | 🔒 |
| **不可穿越方块** | 路径不可穿越任何已有方块（含不同图案、同图案非终点方块） | 🔒 |
| **不可穿越障碍** | 路径不可穿越障碍格（冰块/云/栅栏） | 🔒 |
| **起终点可走** | 路径的起点A和终点B本身算作可走（不算障碍） | 🔒 |

**按折弯数分类的路径检测**

| 折数 | 描述 | 检测方法 |
|------|------|---------|
| 0折（直线） | A和B在同一行或同一列，中间所有格子为空 | 遍历AB之间每格→全空则连通 |
| 1折（L形） | 以拐点C=(Ax,By)或C=(Bx,Ay)为转角，A→C和C→B两段均为直线可通 | 检测两个可能拐点→任一可通则连通 |
| 2折（Z/U形） | 存在两个拐点C1、C2，A→C1→C2→B三段均为直线且全空 | 枚举所有可能的中间行/列→找到一条全空路径则连通 |

**路径搜索算法**

```
路径搜索（round_ray算法）：
1. 优先检测0折（直线连通）→有则返回
2. 检测1折（两个可能拐点）→有则返回
3. 检测2折（枚举中间行/列）→有则返回
4. 全部无→不连通

BFS替代实现：
- 状态=(x, y, 方向, 已折弯次数)
- 扩展规则：沿当前方向前进不折弯 / 转90°折弯次数+1(≤2)
- 碰撞：空格可走，有方块不可走（终点B除外）
- 找到B→记录路径折线 → 消除
```

**行为链 Step3: 下落填补**

| 维度 | 描述 |
|------|------|
| **触发** | 消除动画完成后 |
| **前置检查** | ✅ 被消除格子上方存在方块 |
| **执行** | 每列从下往上扫描：遇到空格则上方所有方块逐个下移，Tween动画下落 |
| **后果** | 方块紧凑到底部，顶部产生新空位 |
| **终止条件** | 所有列下落完成 |
| **显式否定** | ❌ 不生成新方块；❌ 不水平/斜向移动；❌ 空位不填入新图案 |
| **时序约束** | 下落动画300ms(easeOutBounce) |

**行为链 Step4: 死局检测 + 重排**

| 维度 | 描述 |
|------|------|
| **触发** | 每次消除+下落完成后 |
| **前置检查** | ✅ 网格中仍有方块 |
| **执行** | 遍历所有剩余方块对→检查是否存在至少一对可连通消除 |
| **后果-有解** | 继续等待玩家操作 |
| **后果-无解** | 自动重排：交换方块位置强制生成可消除对（最多3次） |
| **终止条件** | 确认有解，或重排次数耗尽→游戏失败 |
| **显式否定** | ❌ 重排不改变图案种类和每种数量，只改位置 |
| **时序约束** | 全盘检测<50ms；重排动画500ms |

**重排规则（🔒核心）**

| 规则项 | 描述 |
|--------|------|
| **保留不变** | 剩余方块的图案种类和每种数量完全不变 |
| **仅重排位置** | 收集所有剩余方块 → Fisher-Yates随机打散 → 重新填入当前有方块的格子 |
| **位置集不变** | 仅在"当前有方块的格子"之间重排，空位保持为空 |
| **最大重排次数** | 每局最多自动重排3次，超出后判定死局→游戏失败 |
| **重排后验证** | 每次重排后必须验证存在至少1对可连通消除；仍无解→继续重排 |
| **重排动画** | 所有方块淡出→位置交换→淡入（500ms），期间锁定输入 |
| **显式否定** | ❌ 不能产生新图案；❌ 不能改变剩余方块总数；❌ 不能填入空位 |

**本游戏为"密铺矩阵"型布局**:
- 16列×10行规则网格，等间距排列

- 消除后空位保留（下落填补后顶部空位不填新方块）
- 不是自由散落型、不是堆叠型、不是六边形型

**与其他布局的区分**:
| 对比项 | 本游戏 | 三消(F04a) | 堆叠(F01a) |

| 网格 | 16×10固定 | 7×7~9×9 | 无固定网格 |

| 路径约束 | ≤2折连通 | 无（相邻即可） | 无（顶层可取） |

#### 关卡生成约束

**可解性保证（🔒核心）**:
1. 每种图案数量必须为偶数（保证理论可全部配对消除）
2. 总方块数 = 网格有效格数（排除障碍物永久占位），必须为偶数
3. 初始布局必须存在至少1对可消除方块
4. 通过`difficulty_control`算法保证：计算剩余方块数 → 强制生成 `floor(remaining / 6)` 对相邻可消除对

**图案分布算法**:
1. 确定本关图案种类数K（8≤K≤12，随关卡递增）
2. 计算总方块数N（N为偶数，N≤160）
3. 将N个方块分配到K种图案，每种至少2个且为偶数
4. Fisher-Yates随机洗牌后填充到网格有效位置
5. 验证初始状态存在可消除对，不满足则重新洗牌

**关卡难度递进**:

| 关卡范围 | 图案种类 | 障碍物 | 初始提示数 |
|---------|---------|--------|-----------|
| 第0关(新手) | 6种 | 无 | 25 |
| 1-10关 | 8种 | 无 | 3 |
| 11-30关 | 10种 | 冰块(少量) | 3 |
| 31+关 | 12种 | 冰块+云/栅栏 | 3 |

### 3.4 关卡系统

#### 模块目标

提供渐进式难度曲线，保持关卡多样性，让玩家有持续推进的目标感。

#### 核心规则

**关卡结构**:
- 网格尺寸：16列×10行（所有关卡固定）
- 关卡配置：JSON数据驱动，包含`MapData`(网格布局)和`block_list`(图案列表)
- 关卡循环：超出配置范围时取模循环（`level_id % config_length`）
- 关卡序号：通关后level严格+1递增

**胜利条件**:
- 网格中所有可消除方块被清空
- 触发通关结算：星星奖励 + 解锁下一关

**失败条件**:
- 死局状态（无可消除对）
- 且自动重排次数耗尽（3次）
- 且无刷新道具

**星星奖励**:
- 基础星星由关卡配置决定
- 存储到 localStorage `highest_score`

#### 数值配置

| 参数 | 值 | 说明 |
|------|-----|------|
| 网格尺寸 | 16×10 | 固定 |
| 最大折弯 | 2 | 固定 |
| 图案种类 | 8-12 | 随关卡递增 |
| 初始提示(Lv0) | 25 | 新手关 |
| 初始提示(Lv>0) | 3 | 普通关 |
| 自动重排上限 | 3次/局 | 固定 |
| 刷新对数 | floor(剩余/6) | 动态 |

#### 显式否定与原型边界

- ❌ 不要实现分支选关（线性推进）
- ❌ 不要实现关卡内计时
- ❌ 不要实现关卡星级评分（1/2/3星），仅通关/失败二元判定
- 关卡进度保存到 localStorage `recent_progress`

#### 模块目标
