## 0.5 Coding Agent 导读

### A. 核心机制摘要
```
🎯 一句话核心：点击底部虚拟按钮控制14×10网格中下落的7种Tetromino方块，
   行填满即消除+重力下落+连锁检测，堆叠触顶则结束。

⚡ 与其他消除类的关键差异：
1. 不是滑动交换三消而是**经典堆叠消除**——方块从顶部下落，填满整行才消除
2. 不是点击/拖拽消除而是**按钮控制移动+旋转**——左移/右移/旋转/硬降四个按钮
3. 不是关卡制而是**无尽生存模式**——无固定关卡，以累计分数为目标
4. 不是匀速下落而是**阶梯加速**——分数越高下落越快(1.0s→0.25s)
5. 不是旋转可踢墙而是**硬碰撞判定**——旋转越界即非法，无Wall Kick补偿
```

### B. 前置否定清单
```
🚫 以下游戏行为绝对不要实现：
❌ 暂存（Hold）功能——不能保存当前方块供后续使用
❌ 幽灵方块（Ghost Piece）投影——不显示方块落地预测位置
❌ 踢墙（Wall Kick）系统——旋转越界即判定非法，不做位移补偿
❌ 连击（Combo）额外加分——仅基础行消除计分，无连击倍率
❌ 软降（Soft Drop）按钮——仅有硬降，无减速下落按钮
❌ 180度快速旋转——仅单向顺时针旋转，无逆旋转
❌ T-Spin等特殊动作加分——不识别特殊旋转技巧
❌ 手势滑动/拖拽控制——只通过底部虚拟按钮操作
⚠️ 包含以上任何一项，说明核心设计理解错误。
```

### C. 反例描述
```
🚨 错误实现特征：

1. ❌ 错误现象：旋转越界时自动偏移方块位置（Wall Kick）
   ✅ 正确现象：旋转后任一瓦片越界或与已锁定方块重叠→旋转不生效，方块保持原状
   📌 验证方法：将I型方块贴紧右墙→点击旋转→方块不旋转，位置不变

2. ❌ 错误现象：方块下落到底部有延迟锁定（Lock Delay），可继续移动
   ✅ 正确现象：方块触底或碰到已锁定方块→立即锁定，不可再操作
   📌 验证方法：方块落到底→立即锁定并生成新方块，无滑动窗口

3. ❌ 错误现象：消除行后上方方块不下落或仅部分下落
   ✅ 正确现象：消除行后上方所有已锁定方块Y坐标+1（整体下移1格）
   📌 验证方法：消除底部一行→上方所有方块均匀下移一行，无空隙残留

4. ❌ 错误现象：消除后不检测新满行（无连锁）
   ✅ 正确现象：重力下落完成后重新扫描全部行→有新满行→继续消除→循环
   📌 验证方法：构造场景使消除后下落产生新满行→应自动连锁消除

5. ❌ 错误现象：新方块生成位置被占时游戏不结束
   ✅ 正确现象：makeShape生成新方块→检测任一瓦片与已锁定方块重叠→立即Game Over
   📌 验证方法：堆叠方块至顶部→新方块生成→应立即显示结算界面

6. ❌ 错误现象：硬降后方块悬停或穿透已锁定方块
   ✅ 正确现象：硬降=循环向下移动直至碰撞→立即锁定在碰撞位置上方
   📌 验证方法：点击硬降按钮→方块瞬间落到最低可达位置并锁定

7. ❌ 错误现象：下落速度不随分数变化
   ✅ 正确现象：分数达到阈值后下落间隔阶梯式缩短(1.0s→0.8s→…→0.25s)
   📌 验证方法：累计分数超过150→下落速度应从1.0s加快到0.8s

8. ❌ 错误现象：初始棋盘已有锁定方块或非空状态
   ✅ 正确现象：开局网格完全清空，仅有一个活动方块从顶部开始下落
   📌 验证方法：开始新游戏→14×10网格全空+顶部生成一个随机方块
```

## 2. 核心循环

```
[游戏开始] → 14×10网格清空 → 随机生成1个Tetromino方块(顶部)
                                    ↓
            ┌──────────── [玩家操作阶段] ←──────────────┐
            │               ↓                           │
            │    自动下落计时器(interval由分数决定)        │
            │               ↓                           │
            │    每tick：方块Y+1(下移1格)                │
            │               ↓                           │
            │    碰撞检测：下方有阻挡?                    │
            │       ├→ 否 → 继续下落 ────────────────────┘
            │       └→ 是 → [锁定方块]
            │                    ↓
            │         方块瓦片加入confirmedTileArray
            │                    ↓
            │         [行消除检测(CHECKING)]
            │               ↓
            │         扫描14行→有满行(10格)?
            │           ├→ 是 → 消除满行 → 重力下落 → 回到CHECKING
            │           └→ 否 → [生成新方块]
            │                        ↓
            │                  生成位置被占?
            │                    ├→ 否 → 回到[玩家操作阶段]
            │                    └→ 是 → [Game Over]
            │
[玩家操作]──┤ 点击左移：方块所有瓦片X-1(碰撞允许时)
            ├ 点击右移：方块所有瓦片X+1(碰撞允许时)
            ├ 点击旋转：方块切换下一旋转态(碰撞允许时)
            └ 点击硬降：循环下移直至碰撞→立即锁定
```

### 核心动词表

| 输入(玩家操作) | 过程(系统响应) | 输出(状态变化) | 反馈(感知) |
|--------------|--------------|--------------|-----------|
| 点击左移 | 碰撞检测→合法则X-1 | 方块左移1格 | 方块平移动画60ms |
| 点击右移 | 碰撞检测→合法则X+1 | 方块右移1格 | 方块平移动画60ms |
| 点击旋转 | 碰撞检测→合法则切换旋转态 | 方块旋转90° | 旋转动画100ms |
| 点击硬降 | 循环下移至碰撞→锁定 | 方块瞬间落底+锁定 | 落地震动+音效 |
| (自动)下落 | 计时器触发→Y+1 | 方块下移1格 | 无特殊反馈 |
| (自动)锁定 | 碰撞→加入锁定数组 | 方块固定 | 落地音效+轻震 |
| (自动)行消除 | 扫描满行→消除→下落 | 分数增加+网格更新 | 行闪白消失+消除音+中震 |
| (自动)连锁 | 下落后再扫描→新满行→继续 | 额外分数 | 连锁文字提示+震屏加强 |

## 3. 系统模块

### 3.1 网格与方块系统

#### 模块目标
管理14×10网格状态，生成、控制和锁定7种标准Tetromino方块，执行行消除与重力下落。

#### 核心规则

**Step1: 方块生成**

| 维度 | 描述 |
|------|------|
| **触发** | 游戏开始 或 上一个方块锁定+行消除完成后 |
| **前置检查** | ✅ 生成位置无已锁定方块占用 |
| **执行** | 随机选择7种Tetromino之一，初始旋转态=0，放置在网格顶部中央(约X=3~4, Y=0) |
| **后果-成功** | 新方块出现，自动下落计时器启动 |
| **后果-失败** | 生成位置被占→Game Over |
| **终止** | 方块出现在网格中 |
| **显式否定** | ❌不做方块预览队列 ❌不做方块随机袋(7-bag)保证 |
| **时序** | 立即生成，无延迟 |

**7种标准Tetromino定义**

| 方块类型 | 代号 | 形状 | 颜色(HEX) | 旋转态数 |
|----------|------|------|-----------|----------|
| I型(Line) | I | ████ (4×1直线) | #00BCD4(青) | 2(水平/垂直) |
| O型(Square) | O | ██/██ (2×2方形) | #FFD700(黄) | 1(不变) |
| T型 | T | ███/·█· | #9C27B0(紫) | 4 |
| S型 | S | ·██/██· | #4CAF50(绿) | 2 |
| Z型 | Z | ██·/·██ | #F44336(红) | 2 |
| J型 | J | █··/███ | #2196F3(蓝) | 4 |
| L型 | L | ··█/███ | #FF9800(橙) | 4 |

**Step2: 玩家操作(移动/旋转/硬降)**

| 维度 | 描述 |
|------|------|
| **触发** | 玩家点击底部虚拟按钮 |
| **前置检查** | ✅ 游戏状态=PLAYING ✅ isPaused==false ⚠️ 动画播放中→锁定输入 |
| **执行-左移** | 所有瓦片X-1，若任一瓦片X<0或与已锁定方块重叠→不执行 |
| **执行-右移** | 所有瓦片X+1，若任一瓦片X>9或与已锁定方块重叠→不执行 |
| **执行-旋转** | 切换下一旋转态坐标，若任一瓦片越界或重叠→不执行(无Wall Kick) |
| **执行-硬降** | 循环Y+1直至碰撞→立即锁定 |
| **终止** | 操作完成或判定非法 |
| **显式否定** | ❌不支持手势滑动 ❌不支持长按连移 ❌不支持拖拽 ❌不支持逆向旋转 |
| **时序** | 移动动画60ms / 旋转动画100ms / 硬降即时 |

**Step3: 自动下落**

| 维度 | 描述 |
|------|------|
| **触发** | 下落计时器到期(interval由当前分数决定) |
| **前置检查** | ✅ 游戏状态=PLAYING ✅ 当前方块未锁定 |
| **执行** | 活动方块所有瓦片Y+1 |
| **后果-无碰撞** | 方块下移，计时器重置 |
| **后果-有碰撞** | 方块无法下移→立即锁定 |
| **终止** | 方块移动完成或触发锁定 |
| **时序** | 见下落速度表 |

**Step4: 锁定与行消除**

| 维度 | 描述 |
|------|------|
| **触发** | 方块碰撞后锁定 |
| **执行** | ①方块瓦片加入confirmedTileArray ②从第0行到第13行逐行扫描 ③满行(该行10格全填)→消除该行所有瓦片 ④上方所有瓦片Y+1(重力下落) ⑤回到②重新扫描直至无满行 |
| **分数** | 每消除1行+100分(推测值) |
| **终止** | 无新满行→生成新方块 |
| **显式否定** | ❌无连击倍率 ❌无T-Spin加分 ❌无四行消除(Tetris)额外奖励 |
| **时序** | 消除动画300ms/行，下落动画50ms/格 |

### 3.2 输入控制系统

#### 模块目标
通过4个虚拟按钮接收玩家输入，转化为方块移动/旋转/硬降操作。

#### 核心规则

| 按钮 | 触发条件 | 判定逻辑 | 执行动作 |
|------|----------|----------|----------|
| 左移 | 点击leftBtn区域 | 检测左移后所有瓦片X≥0且无碰撞 | shapeTileArray所有瓦片X-1 |
| 右移 | 点击rightBtn区域 | 检测右移后所有瓦片X≤9且无碰撞 | shapeTileArray所有瓦片X+1 |
| 旋转 | 点击rotateBtn区域 | 检测旋转后所有瓦片在界内且无碰撞 | 切换至下一旋转态坐标 |
| 硬降 | 点击dropBtn区域 | 无前置检查(立即执行) | 循环moveDown直至碰撞，立即锁定 |

**输入防护**
- 所有输入前检查：`isPaused==false && isLose==false`
- 消除/下落动画播放期间锁定所有输入
- 无长按连移功能：单次点击=单次操作

#### 界面与交互

底部按钮布局（y=1060px起，宽750px）：

```
┌──────────────────────────────────────┐
│                                      │
│   (←)          (↻)    (⬇⬇)    (→)   │
│  x:100       x:320   x:480   x:650  │
│  y:1180      y:1180  y:1180  y:1180  │
│                                      │
└──────────────────────────────────────┘
```

按钮间距：左移与右移分居两侧，旋转和硬降居中偏右。

#### 实现约束
- 按钮触摸响应区域：以按钮中心为圆心，半径48px（大于视觉半径36px，增加容错）
- 按钮按下态：缩放至90% + 亮度降20%，持续100ms
- 多指同时触摸：仅响应第一个触摸点，忽略后续触摸

#### 关键反馈
- 按钮按下：视觉缩放90% + 触感反馈
- 操作成功：方块移动/旋转动画
- 操作非法：方块原地微抖(±3px, 80ms)

#### 显式否定与原型边界
- ❌ 不实现手势滑动控制
- ❌ 不实现点击方块拖拽
- ❌ 不实现长按自动重复
- ❌ 不实现键盘/手柄支持

#### 视觉语义
| 元素 | 默认态 | 按下态 | 不可用态 |
|------|--------|--------|----------|
| 操作按钮 | #4CAF50背景+白色图标，100%大小 | 缩放90%+亮度-20% | #9E9E9E+50%不透明 |

#### 布局类型判定
固定底部栏布局，按钮位置不随游戏状态变化。

#### 关卡生成约束
不适用（输入系统无关卡生成）。

### 3.3 计分与速度系统

#### 模块目标
根据消除行数计算分数，并基于分数阈值调整方块下落速度，形成渐进难度曲线。

#### 核心规则

| 规则 | 描述 |
|------|------|
| 基础计分 | 每消除1行→+100分 |
| 多行消除 | 同时消除N行→N×100分（无额外倍率） |
| 速度调整 | 每次消除完成后检查分数→对应速度表更新下落间隔 |
| 分数显示 | 实时更新顶部状态栏分数文字 |
| 最高分记录 | Game Over时比较当前分数与localStorage中highest_score，取较大值存储 |

#### 难度曲线

| 阶段 | 分数区间 | 下落间隔 | 体验特征 |
|------|----------|----------|----------|
| 入门期 | 0-150 | 1000ms | 宽松节奏，熟悉操作 |
| 适应期 | 150-300 | 800ms | 略微提速，开始需要规划 |
| 进阶期 | 300-450 | 600ms | 明显加快，需要快速决策 |
| 熟练期 | 450-600 | 500ms | 较快速度，考验反应 |
| 高手期 | 600-900 | 400ms | 高速操作，空间规划关键 |
| 专家期 | 900-1500 | 300ms | 极速下落，极限挑战 |
| 极限期 | 1500+ | 250ms | 最高速度封顶，生存竞赛 |

#### 实现约束
- 分数为非负整数，初始值0
- 速度调整在消除动画完成后立即检查执行
- 最高速度250ms为硬上限，不再加速

#### 显式否定与原型边界
- ❌ 无连击(Combo)额外加分
- ❌ 无T-Spin等特殊动作加分
- ❌ 无四行消除(Tetris)额外奖励倍率
- ❌ 无等级/经验系统

### 3.4 游戏结束与结算系统

#### 模块目标
检测游戏结束条件，展示结算界面，提供重新开始功能。

#### 核心规则

| 状态转换 | 触发条件 | 执行动作 |
|----------|----------|----------|
| PLAYING→GAMEOVER | 新方块生成位置被已锁定方块占用 | 停止游戏循环→灰化动画→显示结算UI |
| GAMEOVER→INIT | 点击"再玩一次"按钮 | 调用restart()→清空网格→分数归零→重新开始 |

#### 结算界面布局

```
┌──────────────────────────────────────┐
│           (半透明黑色遮罩)             │
│                                      │
│            挑战失败                    │ 字号48px #FFFFFF
│                                      │
│             450                      │ 字号64px #FFD700
│           本局得分                    │ 字号20px #AAAAAA
│                                      │
│          最高: 1200                   │ 字号28px #FFFFFF
│                                      │
│        [  再玩一次  ]                 │ 160×56px #4CAF50
│                                      │
└──────────────────────────────────────┘
```

#### 实现约束
- 结算弹窗出现前先播放灰化动画(网格从底向顶逐行灰化，500ms)
- 结算弹窗从中心放大弹入(scale 0→1, 300ms, ease-out-back)
- 重玩按钮点击后立即重置，无额外确认

#### 关键反馈
- Game Over触发：网格逐行灰化(从底到顶) + 失败下降音 + 双震(100ms间隔)
- 结算弹窗：弹性动画弹入
- 点击重玩：按钮缩放反馈→画面淡出→新局淡入

#### 显式否定与原型边界
- ❌ 不实现关卡进度/星级评价
- ❌ 不实现复活机制（原型简化，剔除红线外功能依赖）
- ❌ 不实现分数翻倍功能（原型简化）

### 3.5 持久化系统

#### 模块目标
使用localStorage保存最高分和最近进度，实现本地数据持久化。

#### 核心规则
- **highest_score**: 游戏结束时，若当前分数 > 存储值，则更新
- **recent_progress**: 可选，记录最近一局的基本信息（分数、日期等）

#### 实现约束
- 仅允许使用这两个key，禁止额外localStorage操作
- 读写操作使用try-catch包裹，防止存储不可用时崩溃
- 数据格式：highest_score为数字字符串，recent_progress为JSON字符串

#### 显式否定与原型边界
- ❌ 不实现跨设备同步
- ❌ 不实现多存档槽位
- ❌ 不实现存档导入/导出
