## 0.5 Coding Agent 导读

### A. 核心机制摘要

```
🎯 一句话核心：在10×10彩色星星网格中，点击2+相邻同色连通组直接消除→垂直下落→水平向中心聚拢→combo²×5累积分数达标通关

⚡ 与其他消除类的关键差异：
1. 不是滑动交换匹配而是**点击连通组直接消除**——无任何拖拽/交换逻辑
2. 不是3连线判定而是**泛洪四向相邻≥2即可消除**——不要求成线
3. 不是仅垂直下落而是**先垂直下落再水平向中心聚拢**——空列会被两侧列填充
4. 不是固定得分而是**combo²×5连击递增**——连续消除分数指数增长
5. 不是有无限操作而是**方块资源有限不补充**——消除后不生成新方块，只重排已有方块
```

### B. 前置否定清单

```
🚫 以下游戏行为绝对不要实现：
❌ 滑动交换两个方块——玩家只能点击，不存在交换操作
❌ 消除后从顶部生成新方块——方块总量只减不增，下落和聚拢只重排已有方块
❌ 3连线判定——判定条件是泛洪四向相邻连通块≥2，不要求成线
❌ 对角线方向匹配——仅上下左右四向泛洪
❌ L形/T形特殊匹配生成道具——无自动生成特殊道具机制
❌ cascade连锁自动消除——消除+下落+聚拢后回到等待输入，不自动检测新匹配
❌ 物理引擎驱动下落——使用Tween插值动画，非RigidBody
⚠️ 包含以上任何一项，说明核心设计理解错误。
```

### C. 反例描述

```
🚨 错误实现特征：

1. ❌ 错误现象：实现为拖拽交换相邻方块形成三连消除（Match-3模式）
   ✅ 正确现象：点击方块→泛洪检测同色相邻连通组→连通块≥2直接消除
   📌 验证方法：点击一个有≥2同色相邻方块的位置→整个连通组应立即消除

2. ❌ 错误现象：消除后从网格顶部掉落新的随机颜色方块填充空位
   ✅ 正确现象：消除后仅已有方块垂直下落+水平向中心聚拢，不产生新方块
   📌 验证方法：消除一组方块→网格总方块数减少→不出现新方块

3. ❌ 错误现象：水平聚拢方向为统一向左或向右
   ✅ 正确现象：空列出现后，两侧列向画面几何中心滑移
   📌 验证方法：消除中间某列所有方块→左侧列向右移、右侧列向左移→网格保持中心对称紧凑

4. ❌ 错误现象：点击孤立单个方块（无同色相邻）也能消除
   ✅ 正确现象：连通块大小=1时，播放无效抖动反馈，不消除
   📌 验证方法：点击一个四周无同色方块的孤立块→方块抖动→不消除→不扣分

5. ❌ 错误现象：下落和聚拢动画播放期间可以继续点击消除
   ✅ 正确现象：动画播放期间锁定输入（canTouch=false）
   📌 验证方法：消除后快速点击其他方块→不应响应→直到动画结束才恢复输入

6. ❌ 错误现象：消除后下落+聚拢完成后自动检测新匹配并继续消除（cascade）
   ✅ 正确现象：下落+聚拢完成后直接回到等待输入状态，不自动连锁
   📌 验证方法：消除一组后下落聚拢完成→游戏等待玩家下一次点击→不自动消除

7. ❌ 错误现象：初始网格存在孤立方块无法消除导致立即死局
   ✅ 正确现象：初始生成的网格必须保证至少存在一个≥2的同色相邻连通组
   📌 验证方法：开局→检查网格→至少有一处可点击消除
```

```

      ↓                                         │

**核心动词**: 点击、消除、下落、聚拢

| 输入(玩家操作) | 过程(系统响应) | 输出(状态变化) | 反馈(感知) |

## 3. 系统模块

### 3.1 点击消除系统

#### 模块目标

玩家通过点击同色相邻连通组消除方块获得分数，利用连击机制追求高分。

#### 核心规则

**Step1: 点击检测**

| 维度 | 描述 |
|------|------|
| **触发** | 手指点击（TOUCH_START）网格内方块 |
| **前置检查** | ✅ canTouch=true（非动画中） ✅ 点击位置命中方块 ❌ 动画播放中→忽略 |
| **执行** | 从点击坐标出发，四向泛洪收集同色相邻方块组成连通块 |
| **后果-成功** | 连通块≥2→进入消除流程 |
| **后果-失败** | 连通块=1→方块抖动反馈→回到等待输入 |
| **终止** | 泛洪完成返回连通块列表 |
| **显式否定** | ❌不支持对角线方向 ❌不支持拖拽/滑动 ❌不支持多指 |
| **时序** | 泛洪检测<16ms（同帧完成） |

**Step2: 消除+计分**

| 维度 | 描述 |
|------|------|
| **触发** | 泛洪返回连通块≥2 |
| **前置检查** | ✅ 连通块列表非空 |
| **执行** | 连通块所有方块播放缩放消失动画→从gridList标记为null→计分combo²×5→combo+1 |
| **终止** | 消除动画完成(200ms) |
| **时序** | 消除动画200ms |

**Step3: 垂直下落**

| 维度 | 描述 |
|------|------|
| **触发** | 消除动画完成 |
| **前置检查** | ✅ 网格中存在空位 |
| **执行** | 每列从底向上扫描，上方方块垂直下落到该列最低空位 |
| **终止** | 所有方块落稳 |
| **时序** | 每格50ms，ease-out缓动 |

**Step4: 水平聚拢**

| 维度 | 描述 |
|------|------|
| **触发** | 垂直下落完成 |
| **前置检查** | ✅ 存在空列 |
| **执行** | 所有列向画面几何中心水平滑移，消除中间空列 |
| **终止** | 聚拢动画完成 |
| **显式否定** | ❌不是统一向左或向右，而是双向向中心 |
| **时序** | 整列平移200ms |

**Step5: 状态检测**

| 维度 | 描述 |
|------|------|
| **触发** | 聚拢动画完成 |
| **执行** | 检查currScore≥aimScore→通关；否则全盘检测是否存在≥2同色相邻连通块 |
| **后果-通关** | 进入通关结算（剩余方块转奖励分） |
| **后果-死局** | currScore<aimScore→失败界面 |
| **后果-继续** | canTouch=true→回到等待输入 |

**连击(combo)机制**:
- 每次成功消除combo+1
- 得分公式：当次得分 = combo² × 5
- combo在连击超时（约2-3秒无操作）后重置为0
- 连击等级文字提示："好"(combo≤3)/"酷"(4-6)/"太棒了"(7-9)/"非常完美"(10-14)/"难以置信"(15+)

#### 显式否定与原型边界

```
🚫 绝不做：
1. ❌ 滑动交换——只能点击
2. ❌ 消除后生成新方块——方块总量只减不增
3. ❌ cascade自动连锁——消除+下落+聚拢后等待输入
4. ❌ 对角线匹配——仅四向(上下左右)
5. ❌ L/T/十字形特殊匹配——无自动特殊道具生成

- 剩余方块奖励分具体倍率：建议每剩余方块20分，可后期调整

- **普通方块**: 58×58px圆角正方形，纯色填充+中央白色五角星图案(星星直径约24px)，5色区分
- **方块间隙**: 2px深色间隙线(#0a0e30)，形成网格感
- **选中高亮**: 点击瞬间被选中的连通组方块叠加rgba(255,255,255,0.3)白色半透明层，持续至消除动画开始
- **下落轨迹**: 无拖尾，方块以ease-out缓动直接移动到目标位置
- **空列标识**: 无显式标识，聚拢动画自然消除空列

#### 布局类型判定

- **类型**: 固定尺寸规则网格（10×10）
- **特征**: 方块等大、无重叠、无层叠遮挡、紧密平铺
- **与F04a-Match3的区别**: 无顶部生成口（方块不补充），网格随游戏进行逐渐缩小（方块减少+聚拢）
- **与F04b-PopStar的对齐**: 完全对齐——点击消除+垂直下落+水平聚拢+不补充新方块

#### 关卡生成约束

| 约束 | 要求 |
|------|------|
| 初始可玩 | 生成后检测是否存在至少1个≥2同色相邻连通组，不存在→重新生成 |
| 颜色分布 | 5种颜色按LVConfig权重随机分配，使用OutOfOrder算法打乱 |
| 网格尺寸 | 固定10×10，100个方块 |
| 颜色种类 | 固定5种（紫/红/蓝/绿/橙） |
| 难度分段 | Level≤10使用配置0, ≤50使用配置1, ≤100使用配置2（影响颜色权重） |
| 死局处理 | 全盘检测无≥2同色相邻连通组→触发失败结算（不自动洗牌，与Match-3不同） |

### 3.2 物理补位系统

#### 模块目标

消除后通过垂直下落和水平聚拢保持网格紧凑，为后续消除提供操作空间。

#### 核心规则

**垂直下落 (_checkGridFall)**:
1. 遍历每列，从底部向上扫描
2. 遇到空位时，找到上方最近的非空方块
3. 该方块以Tween动画垂直下移到空位
4. 重复直到该列无空隙

**水平聚拢 (_checkGridHorMove)**:
1. 下落完成后，检测是否存在空列（整列无方块）
2. 空列左侧的所有列向右滑移、空列右侧的所有列向左滑移，方向为画面几何中心
3. 消除空列，保持网格紧凑对称

#### 实现约束

- 动画类型：Tween位置插值，非物理引擎
- 方块之间无碰撞体积
- 下落速度：50ms/格，ease-out
- 聚拢速度：200ms/次，ease-in-out
- 执行顺序：必须先下落完成→再聚拢，不可并行
- 聚拢完成后才允许下一次输入(canTouch=true)

#### 显式否定

- ❌ 不使用真实重力加速度
- ❌ 无碰撞反弹效果
- ❌ 无方块旋转动画
- ❌ 不从顶部生成新方块

### 3.3 计分与连击系统

#### 模块目标

通过combo递增计分激励玩家连续快速消除，制造策略与紧迫的张力。

#### 核心规则

**计分公式**:
```
单次消除得分 = combo² × 5
```
- combo从0开始，每次成功消除后combo+1
- 第1次消除: 1²×5=5分
- 第2次消除: 2²×5=20分
- 第3次消除: 3²×5=45分
- 第10次消除: 10²×5=500分

**连击超时**:
- 若2-3秒内无新消除操作，combo重置为0
- 下一次消除从combo=1重新开始

**剩余方块奖励**:
- 通关时，剩余方块数量×20分作为额外奖励
- 方块越少说明消除越彻底，但不额外奖励

**combo文字等级**:

| combo范围 | 显示文字 | 文字颜色 |
|----------|---------|---------|
| 1-3 | "好" | #FFEB3B |
| 4-6 | "酷" | #FF9800 |
| 7-9 | "太棒了" | #FF5722 |
| 10-14 | "非常完美" | #F44336 |
| 15+ | "难以置信" | #E91E63 |

#### 实现约束

- combo为全局变量，关卡内持续累积
- 超时计时器在每次消除后重置
- 通关结算时先计算剩余方块奖励再显示总分

### 3.4 关卡系统

#### 模块目标

通过递增的目标分数提供持续挑战，关卡无限。

#### 核心规则

**关卡目标分数 (aimScore)**:

| 关卡 | 目标分数 | 计算方式 |
|------|----------|---------|
| 1 | 1000 | 固定 |
| 2 | 3000 | 固定 |
| 3 | 4500 | 固定 |
| 4 | 7500 | 固定 |
| 5 | 10000 | 固定 |
| 6+ | 12700 + 60×(L-6) | 线性公式 |

**通关判定**:
- currScore ≥ aimScore → 通关 → 奖励5星星 → 下一关

**失败判定**:
- 全盘无≥2同色相邻连通组 且 currScore < aimScore → 失败

**难度缩放 (LVConfig)**:
- Level 1-10: 配置0（颜色权重均匀，较易形成大连通组）
- Level 11-50: 配置1（权重微调，增加散布）
- Level 51-100: 配置2（权重进一步分化，更难形成大组）
- Level 100+: 继续使用配置2

#### 实现约束

- 关卡数据优先从预设配置加载，缺失则回退随机生成
- 每关开始时combo重置为0
- currScore每关重置为0
- 网格固定10×10，不随关卡变化

---

### 3.5 道具系统

#### 模块目标

提供策略工具帮助玩家突破困局，库存制消耗。

#### 核心规则

| 道具 | 功能 | 使用方式 | 效果 |
|------|------|---------|------|
| 锤子(Delete) | 消除指定单个方块 | 点击道具→点击目标方块 | 目标方块消失→触发下落聚拢 |
| 同色消除(DeleteSame) | 消除全屏指定颜色所有方块 | 点击道具→点击某颜色方块 | 该颜色所有方块消失→触发下落聚拢 |
| 染色(Color) | 改变目标方块颜色 | 点击道具→点击目标→点击源颜色方块 | 目标变为指定颜色 |
| 全盘刷新(ChangeAll) | 重新随机生成所有方块颜色 | 点击道具立即执行 | 全部方块重新随机着色→保证可玩 |

**库存管理**:
- 初始数量：每种1个
- 通关奖励：每关完成可能随机补充1个
- 库存为0时按钮显示灰色禁用态
- 道具效果直接修改网格数据，不经过匹配判定
- 使用道具后触发正常的下落和聚拢流程

#### 实现约束

- 道具使用不消耗combo，不影响combo计数
- 道具效果的消除不计入combo得分（或按combo=0计分，即0分）
- 道具动画期间锁定输入
- 同色消除道具的消除动画：从点击位置向四周扩散波纹，逐个消失，总时长400ms
- 全盘刷新动画：所有方块同时旋转360°+颜色变化，时长300ms

### 3.6 复活与失败系统

#### 模块目标

在死局时给予玩家有限的二次机会。

#### 核心规则

**失败流程**:
1. 检测到死局（无可消除连通组 + 未达标）
2. 弹出失败面板
3. 选项A：消耗20星星复活→重置棋盘（重新生成全部方块），保留当前分数，继续当前关卡
4. 选项B：放弃→回到主界面

**星星获取**:
- 通关奖励：每关5星星
- 初始赠送：建议50星星（够2.5次复活）

#### 实现约束

- 复活时重新生成10×10满铺网格，但currScore保留
- 复活不重置combo（combo在超时后自然重置）
- 星星不足时复活按钮显示灰色禁用+显示当前星星数

---

### 3.7 DDA（动态难度调节）

#### 模块目标

防止玩家连续失败流失，微调难度保持体验。

#### 核心规则

- 连续失败3次同一关：下次生成时颜色权重微调，增加同色相邻概率（具体为主色权重+10%）
- 前5关（教学期）：生成时保证至少3个≥4的大连通组
- DDA调整不影响aimScore，仅影响初始网格颜色分布

## 7. 硬性边界合规声明

本文档仅包含核心玩法与本地进度，不包含红线外系统。

**localStorage 使用限制**:
- 仅允许保存两个 key：`highest_score` 与 `recent_progress`
- `highest_score`: number，历史最高分
- `recent_progress`: JSON字符串，包含 `{currLevel, star, propDelete, propDeleteSame, propColor, propChangeAll}`

**运行形态**: 纯本地单机核心玩法体验，无需任何网络连接。
