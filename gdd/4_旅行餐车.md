## 0.5 Coding Agent 导读

### A. 核心机制摘要

🎯 一句话核心：玩家在二维网格内自由拖拽，将两个相同ID物品合并为下一级物品（Merge-2），管理有限空间完成关卡目标。

⚡ 与常见模式的关键差异：
1. 不是三消（Candy Crush式滑动交换+连锁下落），而是两个相同物品拖拽重叠直接升级，无cascade
2. 不是Merge-3（3合1），而是Merge-2（2合1），合成判定基于itemid相同且存在mergeid配置
3. 不是物理堆叠（合成大西瓜式重力碰撞），而是固定网格内离散位置的拖拽操作
4. 不是卡槽收集（羊了个羊式底部槽），网格内直接合成，无底部收集区
5. 拖拽到不同ID物品上是交换位置，不是无效操作弹回——交换与合成共用拖拽手势

### B. 前置否定清单

🚫 以下游戏行为绝对不要实现：
❌ 底部7格收集槽/卡槽——玩家不应看到屏幕底部有收集等待区
❌ 三消式cascade下落补位——合成后不应有上方物品自动下落填充
❌ 物理碰撞/重力模拟——物品不应有自由落体或碰撞弹跳行为
❌ 方向键/摇杆/瞄准线——输入模式为直接拖拽，不应有方向指示器
❌ Merge-3判定（需要3个才能合成）——2个相同即可合成
❌ 2048式整行/整列滑动——物品逐个拖拽，不做整体推动
❌ 自动寻路/自动合成——玩家必须手动拖拽每一次合成
⚠️ 包含以上任何一项，说明核心设计理解错误。

### C. 反例描述

🚨 错误实现特征：

1. ❌ 错误现象：拖拽物品A到同ID物品B上，两个物品都消失并在原位生成新物品
   ✅ 正确现象：物品A消失，物品B原地升级为mergeid对应的下一级物品（仅B位置有物品）
   📌 验证方法：拖拽两个1级物品合成 → 观察到源位置变空、目标位置出现2级物品，总物品数减1

2. ❌ 错误现象：合成需要3个相同物品才能触发
   ✅ 正确现象：仅需2个相同ID物品即可合成
   📌 验证方法：在网格中放置恰好2个同ID物品 → 拖拽其中一个到另一个上 → 合成成功

3. ❌ 错误现象：拖拽物品到不同ID物品上，操作无效，物品弹回原位
   ✅ 正确现象：拖拽到不同ID物品上执行位置交换（Exchange）
   📌 验证方法：拖拽1级物品到2级物品上 → 两者互换位置，均保持原样

4. ❌ 错误现象：合成后上方物品自动下落填充空位（cascade）
   ✅ 正确现象：合成后源位置保持空格，不触发任何自动下落
   📌 验证方法：合成中间行的物品 → 观察上方物品不移动，空格保持空

5. ❌ 错误现象：锁定格子上的物品可以被拖拽或被合成覆盖
   ✅ 正确现象：lock=1的格子完全不可交互，不能拖出也不能拖入
   📌 验证方法：尝试拖拽物品到锁定格子 → 操作被拒绝，物品弹回

6. ❌ 错误现象：动画播放期间玩家可以继续拖拽其他物品
   ✅ 正确现象：合成动画播放期间全局输入锁定（LockMove），动画结束后解锁
   📌 验证方法：合成触发后立刻尝试拖拽另一物品 → 无响应，等动画结束后才可操作

7. ❌ 错误现象：关卡初始生成的网格中存在无法通过任何合成链到达目标物品的孤立物品
   ✅ 正确现象：初始物品配置保证所有物品都能参与合成链，且目标物品可达
   📌 验证方法：检查初始生成 → 每种物品至少出现偶数个（或通过掉落补充），合成链完整覆盖目标

## 2. 核心循环

### Q1: 玩家的一次完整操作是什么？

**操作类型**: 自由拖拽（跟手移动）

**完整操作链**:
1. **触发**: 玩家在网格上按住一个物品（MOUSE_DOWN/TOUCH_START）
2. **选中反馈**: 物品放大至1.1倍，出现#FFD700高亮边框，播放轻点击音效
3. **拖动**: 物品跟随手指移动，原位置保留α=0.3半透明影子
4. **悬停判定**: 拖拽经过同ID物品时，目标出现绿色脉冲光圈（#4CAF50，300ms周期）
5. **释放**: 玩家松手（MOUSE_UP/TOUCH_END）
6. **结果判定**:
   - 目标为同ID物品且存在mergeid → 执行合并（源消失，目标升级）
   - 目标为空格子 → 执行移动（物品滑入空格）
   - 目标为不同ID物品 → 执行位置交换（Exchange）
   - 目标为锁定格子或无效区域 → 取消操作，物品弹回原位（弹性缓动200ms）

**约束条件**:
- 只能拖拽未锁定的物品
- 锁定格子（lock=1）不能作为拖拽目标
- 动画播放期间（LockMove状态）全局输入锁定

### Q2: 合成的判定条件是什么？

**合成判定条件**（全部满足才执行）:
1. 源物品与目标物品itemid相同
2. 目标物品存在有效的mergeid配置（指向下一级物品）
3. 目标格子未被锁定（lock=0）
4. 当前不在LockMove状态（无动画播放中）

**成功合成后**:
- 删除源物品实例（源格子变空）
- 目标物品ID更新为mergeid对应值（原地升级）
- 播放合成特效（金色粒子扩散+缩放弹跳动画300ms）

### Q3: 合成之后发生什么？

**合成后连锁反应**（按顺序执行）:

1. **掉落检测**（checkDrop）:
   - 当合成产物等级≥5时，根据配置概率reward_drop
   - 在网格随机空位生成低级物品（掉落动画：从上方滑入200ms）

2. **气泡产出**（概率触发）:
   - 合并计数累积达到阈值时触发
   - 在空位生成气泡物品（最多5个，mBubbleMax=5）
   - 气泡以弹跳方式出现（缩放0→1.2→1.0，400ms）

3. **任务检查**（later_checks）:
   - 检查合成产物是否匹配当前关卡目标
   - 若匹配则更新进度条

4. **胜利检测**:
   - 所有关卡目标完成 → 进入胜利结算

5. **死局检测**（checkDeadlock）:
   - 网格已满且无任何可合成对 → 提示玩家出售或移入背包

### Q4: 失败条件是什么？

**主玩法无严格失败**，而是死局/停滞状态:
- 网格被填满且无任何可合成对
- 玩家可通过以下方式破局:
  - 出售低级物品腾出空间
  - 将物品移入背包（背包内物品数≥3时才可移出）
  - 等待气泡/掉落产出

### Q5: 核心张力是什么？

**空间管理 vs 合成升级**:
- 有限网格空间要求玩家决策保留/出售哪些物品
- 低级物品是合成基础但占用空间
- 高级物品价值高但需要多个低级物品

**即时收益 vs 长期规划**:
- 出售物品可立即获得金币但失去合成材料
- 保留物品可达成高级合成但面临空间压力

## 3. 系统模块

### 3.1 网格合成系统

#### 模块目标

玩家在6×6网格内通过Merge-2拖拽合成升级物品，完成关卡指定的目标物品合成任务。

#### 核心规则

**行为链展开——拖拽合成**:

| 维度 | 描述 |
|------|------|
| **触发条件** | 玩家在非锁定物品上按下（MOUSE_DOWN）且当前LockMove=false |
| **输入** | 手指/鼠标按住并拖动 |
| **过程** | 1.选中放大1.1倍+高亮 → 2.跟手移动+原位留影 → 3.悬停检测目标类型 → 4.释放执行判定 |
| **判定规则** | 同ID+有mergeid→合成；不同ID→交换；空格→移动；锁定/无效→弹回 |
| **输出/结果** | 合成：源消失+目标升级；交换：互换位置；移动：物品到新位置 |
| **反馈** | 合成：金色粒子+升级弹跳300ms+音效；交换：双向滑动200ms；弹回：弹性缓动200ms |
| **异常处理** | 拖拽超出网格边界→弹回；动画中操作→忽略输入 |
| **状态变更** | 合成后：源cellindex清空，目标itemid=mergeid值；掉落/气泡产出可能改变其他格子 |

**行为链展开——掉落产出**:

| 维度 | 描述 |
|------|------|
| **触发条件** | 合成产物等级lv≥5，且reward_drop概率命中 |
| **输入** | 自动触发，无玩家输入 |
| **过程** | 1.查找网格空位(getEmptyCell) → 2.随机选择一个空位 → 3.生成低级物品 |
| **判定规则** | 有空位→生成；无空位→跳过 |
| **输出/结果** | 空位出现新物品 |
| **反馈** | 物品从格子上方滑入，200ms缓出动画 |
| **异常处理** | 无空位时静默跳过 |
| **状态变更** | 目标格子cellindex填入新itemid |

**行为链展开——气泡产出**:

| 维度 | 描述 |
|------|------|
| **触发条件** | 合并计数累积达到阈值，且当前气泡数<5(mBubbleMax) |
| **输入** | 自动触发 |
| **过程** | 1.检查mBubbleCt计数 → 2.查找空位 → 3.生成气泡物品(box=4) |
| **判定规则** | 计数达标+气泡<5+有空位→生成 |
| **输出/结果** | 空位出现气泡物品，点击可收取 |
| **反馈** | 弹跳出现动画（缩放0→1.2→1.0，400ms） |
| **异常处理** | 达上限或无空位→跳过 |
| **状态变更** | mBubbleCt重置，目标格子更新 |

#### 显式否定与原型边界

**不做清单（游戏体验层面）**:
- ❌ 不做三消式cascade下落——合成后空位保持空
- ❌ 不做批量合成——单次仅1对1合并
- ❌ 不做自动合成/自动寻路——完全手动拖拽
- ❌ 不做斜向拖拽判定——拖拽为自由方向但目标判定基于中心点所在格
- ❌ 不做跨关卡拖拽——网格独立
- ❌ 不做Merge-3（3合1）——始终Merge-2
- ❌ 不做2048式整行滑动

**原型简化范围**:
- 网格固定6×6，不实现动态扩展
- 合成链3-5级，不实现完整长链
- 物品用纯代码绘制（圆角矩形+颜色区分），不依赖外部美术
- 无体力限制，无限操作
- 无活动副本

#### 视觉语义

**布局验证的视觉特征**:
- 网格必须呈正方形格子矩阵，水平垂直对齐
- 物品必须中心对齐于格子内，不溢出
- 等级标签必须在物品右上角可见
- 锁定格子必须有明显云雾覆盖遮挡物品
- 选中高亮必须在拖拽开始时立即出现
- 拖拽时原位影子与跟手物品必须同时可见

#### 布局类型判定

**本游戏属于"固定网格型"布局**:
- 所有物品位于离散网格位置，无自由坐标
- 网格尺寸固定（原型6×6），单元格等大
- 物品不堆叠（每格最多1个），无物理层叠
- 与"自由放置型"（如合成大西瓜）的区别：物品自动吸附到格子中心

#### 关卡生成约束

**可解性保证**:
1. 初始物品中，每种itemid必须出现偶数个（2或4），确保可两两合成
2. 合成链覆盖验证：从初始物品通过合成链必须能到达目标物品
3. 空位保证：初始填充率不超过90%（6×6网格至少留3-4个空格）

**算法要求**:
1. 确定关卡目标物品及其合成链
2. 逆向分解：目标物品 → 所需的前级物品对 → 所需的更前级物品对...
3. 将分解出的最底级物品随机放入网格
4. 补充额外物品（成对放入）增加复杂度
5. 验证：初始状态下至少存在1个可合成对

**死锁预防**:
- 初始生成确保至少3个可合成对
- 掉落产出机制提供额外物品来源
- 出售和背包机制提供空间释放途径

### 3.2 背包系统

#### 模块目标

为玩家提供网格外的物品存储空间，缓解网格空间压力，支持物品管理策略。

#### 核心规则

| 规则项 | 实现 |
|--------|------|
| 初始容量 | 25格 |
| 布局 | 5列×5行网格 |
| 存入条件 | 网格物品未锁定，背包有空位 |
| 取出条件 | 背包内物品总数≥3个时才可移出（"道具不足3个,无法移出"） |
| 排序规则 | 按itemid升序 → time降序 → type排序 |
| 扩容 | 消耗钻石，每次+5格（原型中最多扩容2次，上限35格） |

#### 界面与交互

**背包弹窗ASCII线框**:

```

│  ┌────┬────┬────┬────┬────┐     │
│  │ 🟢 │ 🔵 │ 🟠 │    │    │     │ ← 5列网格
│  ├────┼────┼────┼────┼────┤     │   每格100×100px
│  │ 🔴 │ 🟢 │    │    │    │     │
│  ├────┼────┼────┼────┼────┤     │
│  │    │    │    │    │    │     │ ← 空格虚线边框
│  ├────┼────┼────┼────┼────┤     │

│  ├────┼────┼────┼────┼────┤     │
```

**交互流程**:
1. 点击主界面"仓库"按钮 → 弹窗打开（遮罩淡入200ms+弹窗从底部滑入300ms）
2. 点击背包内物品 → 选中高亮（同网格选中样式）
3. 点击"取出" → 物品移至网格空位（飞行动画300ms）
4. 从网格长按物品+拖拽到仓库按钮 → 物品移入背包
5. 点击"整理" → 物品按排序规则重排（物品滑动到新位置200ms）
6. 点击关闭/遮罩 → 弹窗关闭（反向动画200ms）

#### 实现约束

**弹窗尺寸**: 525×1067px（屏幕70%宽×80%高），居中，圆角20px
**遮罩**: 全屏#000000(α=0.6)
**格子尺寸**: 每格100×100px，间距4px
**物品显示**: 占格子70%面积，约70×70px

**容量检测逻辑**:
- isFull(): 遍历背包数组，若所有itemid≠0则返回true
- 存入前检查isFull()，满则提示"仓库已满"
- 取出前检查物品总数<3则提示"道具不足3个,无法移出"

#### 关键反馈

| 事件 | 视觉反馈 | 音效 |
|------|----------|------|
| 存入成功 | 物品缩小飞入仓库按钮位置 | 放入音 |
| 取出成功 | 物品从弹窗飞出到网格空位 | 取出音 |
| 仓库已满 | 弹窗抖动+红色边框闪烁 | 警告音 |
| 道具不足3个 | Toast提示文字，1500ms后消失 | 提示音 |

#### 显式否定与原型边界

- ❌ 不做背包内合成——背包仅存储，合成只在网格内进行
- ❌ 不做物品自动存入背包——必须手动操作
- ❌ 不做背包物品过期机制

#### 视觉语义

- 背包弹窗为白色底、深色遮罩，视觉层级明确高于网格
- 空格用虚线边框(#E0E0E0 dashed 1px)区分已占用格
- 扩容格用锁图标(#FFC107)标识

#### 布局类型判定

背包为标准5列网格弹层，非滚动列表，非自由放置。

#### 关卡生成约束

背包不参与关卡生成。背包在新关卡开始时保留上一关内容（存入recent_progress）。

### 3.3 任务目标系统

#### 模块目标

#### 界面与交互

```
  目标: [🟣×2] [🔴×1]    进度: 1/2  0/1
```

#### 实现约束

- 任务面板固定在网格区域上方，高度40px

- ❌ 不做多类型任务（仅合成目标，不做收集/时间限制等）

#### 视觉语义

- 目标图标与网格内物品使用相同配色，玩家能直观对应

#### 关卡生成约束

- 目标物品必须在当前关卡合成链可达范围内
- 目标数量≤3种物品，每种数量≤3个

## 4. 资源与经济

### 资源类型

| 资源 | 用途 | 获取方式 | 消耗方式 |
|------|------|----------|----------|
| 星星(Star) | 关卡进度标记 | 完成关卡奖励(3-10个) | 解锁新关卡(原型中不消耗) |
| 金币(Gold) | 通用货币 | 出售物品 | 背包扩容 |
| 钻石(Diamond) | 高级货币 | 完成特殊目标关卡(5-20个) | 背包扩容（原型：50钻石/次） |

### 经济循环

```
  合成物品 ─→ 完成关卡目标 ─→ 获得星星+金币
     ↑                              ↓
  掉落/气泡 ←── 继续游戏 ←── 解锁新关卡
     ↑                              ↓
  出售物品 ←── 空间不足 ←── 更高难度关卡
     ↓
  获得金币 ─→ 背包扩容 ─→ 更多存储空间
```

**闭环验证**:
- 金币产出：出售物品（持续产出）
- 金币消耗：背包扩容（有限消耗）
- 星星产出：关卡奖励（持续产出）
- 钻石产出：特殊关卡（低频产出）
- 钻石消耗：背包扩容（有限消耗）

## 6. 视觉与审美

### 6.1 整体风格

## 7. 硬性边界合规声明

本文档仅包含核心玩法与本地进度，不包含红线外系统。

所有数据仅通过 localStorage 在本地保存，使用两个 key：
- `highest_score`：最高分/最高星星数记录
- `recent_progress`：最近游戏进度（当前关卡编号、网格状态快照、背包内容快照）

本产品为纯本地单机核心玩法体验。
