## 0.5 Coding Agent 导读

### A. 核心机制摘要

🎯 一句话核心：点击动物使其沿当前朝向直线冲刺，冲出棋盘边界即消除，在倒计时内清空所有目标动物。

⚡ 与常见模式的关键差异：
1. **不是三消/交换**，而是**点击触发定向冲刺**——动物沿固定朝向(0°/90°/180°/270°)直线移动，无拖拽、无滑动交换
2. **不是碰撞连锁**，而是**碰撞即停**——动物撞到其他实体立即停止，无推动、无动量传递、无连锁反应
3. **不是底部收集槽消除**，而是**冲出边界消除**——动物移出网格边界即算成功逃离，无卡槽、无容器
4. **不是步数限制**，而是**时间限制+死局检测**双失败条件
5. **所有动物统一占 2×2 格**，无 1×1 小单位

### B. 前置否定清单

🚫 以下游戏行为绝对不要实现：
- ❌ 底部收集槽/卡槽区——动物冲出棋盘边界即消除，不进入任何容器
- ❌ 拖拽/滑动交换操作——玩家只能单指点击，点击后动物自动沿朝向移动
- ❌ 碰撞后推动或连锁反应——碰撞即停，被撞实体不动
- ❌ 重力下落/cascade 补位——动物消除后不触发下落填充
- ❌ 颜色匹配消除——动物按类型区分，无颜色配对机制
- ❌ 3D 视角旋转——固定 2D 正交俯视视角
- ❌ 步数限制——失败条件为时间耗尽或死局，无步数计数

⚠️ 包含以上任何一项，说明核心设计理解错误。

### C. 反例描述

🚨 错误实现特征：

1. ❌ 错误现象：点击动物后弹出方向选择 UI，玩家选择方向后动物才移动
   ✅ 正确现象：动物有固定朝向（由关卡配置决定），点击后立即沿当前朝向移动，无方向选择
   📌 验证方法：加载任意关卡 → 点击一只朝右的动物 → 动物应立即向右直线冲刺，无弹窗/无方向盘

2. ❌ 错误现象：动物 A 撞到动物 B 后，B 被推动一格
   ✅ 正确现象：A 碰撞 B 后 A 立即停止，B 保持原位不动
   📌 验证方法：让一只朝右的动物冲向另一只静止动物 → 移动动物在碰撞点停止 → 被撞动物位置不变

3. ❌ 错误现象：动物到达棋盘边界后停在边缘最后一格
   ✅ 正确现象：动物前方无障碍且朝向边界时，冲出棋盘、播放逃离动画、节点销毁、winCount++
   📌 验证方法：将一只朝向边界的动物前方清空 → 点击该动物 → 动物滑出棋盘 → 进度计数+1

4. ❌ 错误现象：动物消除后上方动物下落填充空位
   ✅ 正确现象：消除后原位置变为空格，其余动物保持原位不动
   📌 验证方法：消除一只位于棋盘中部的动物 → 观察周围动物 → 全部保持原位，无下落

5. ❌ 错误现象：移动过程中可以点击另一只动物使其同时移动
   ✅ 正确现象：`is_moveing` 互斥锁确保同一时刻只有一只动物在移动，移动中点击被忽略
   📌 验证方法：点击动物 A 使其移动 → 在 A 移动动画未结束时点击动物 B → B 不响应

6. ❌ 错误现象：关卡生成时所有动物朝向相同
   ✅ 正确现象：关卡配置中每只动物有独立的 `face` 值（0°/90°/180°/270°），朝向各异
   📌 验证方法：加载关卡 → 观察棋盘上动物 → 应存在至少 2 种不同朝向

7. ❌ 错误现象：关卡生成时存在无解局面（所有动物被完全堵死且无道具）
   ✅ 正确现象：关卡初始状态至少有一只动物可移动（getMovableAnimals 非空），或提供道具破局
   📌 验证方法：加载关卡 → 检查 getMovableAnimals() → 返回数组长度 > 0

## 1. 游戏概述

### 1.1 游戏类型

- **机制族**: F05a（定向逃离）
- **一句话核心循环**: 点击 → 沿朝向冲刺 → 冲出边界消除 → 清空棋盘过关
- **核心动词**: 点击、规划、疏通
- **体验关键词**: 空间规划、路径疏通、时间紧迫、连续消除的爽快感、解谜成就感
- **视觉风格**: 低多边形卡通农场，明快色彩，圆润造型
- **交互模式**: 单指点击（非拖拽、非滑动）

### 1.2 核心目标

玩家在倒计时结束前，通过合理规划点击顺序，将棋盘上所有目标动物移出边界。每只动物有固定朝向（上/下/左/右），点击后沿朝向直线冲刺，被其他实体阻挡则停止，冲出棋盘则消除计数。核心策略在于：移动 A 动物可能为 B 创造通路，也可能堵塞 C 的逃离路径。

## 2. 核心循环

### 一局完整体验（玩家视角）

1. **进入关卡**：棋盘上排满了朝向各异的小猪和少量障碍物，顶部显示进度（0/N）和倒计时
2. **观察棋盘**：找到朝向边界且前方无阻挡的动物——这是最直接的消除目标
3. **点击动物**：动物沿朝向直线冲出，冲出边界则消除（进度+1），被挡则停在碰撞点
4. **规划路径**：一只动物消除后，原本被它挡住的其他动物可能获得新的逃离路径
5. **使用道具**（可选）：遇到死局时使用洗牌重置朝向 / 移除特定阻碍 / 翻转朝向
6. **过关或失败**：所有目标动物消除 → 胜利；时间耗尽或死局无道具 → 失败

### 核心张力

**空间规划 vs 时间压力**：动物间的阻挡关系构成策略网——每次点击都会改变全局阻挡状态。玩家需要在时间限制下快速判断最优移动顺序。

### 完整状态机

```
[INIT] 关卡初始化
  ├── 加载关卡配置（动物位置、朝向、winLength）
  ├── 生成动物实体，设置网格占用
  ├── 初始化 winCount=0，启动倒计时
  └── → [IDLE]

[IDLE] 等待输入 ★主循环状态★
  └── 玩家点击 → [VALIDATE]

[VALIDATE] 输入校验
  ├─ is_moveing=true → 忽略，返回 [IDLE]
  ├─ 道具模式激活（移除/翻转） → 执行道具效果 → 返回 [IDLE]
  ├─ 动物前方被阻挡（不在 getMovableAnimals 列表） → 忽略，返回 [IDLE]
  └─ 校验通过 → [MOVE]

[MOVE] 执行移动
  ├── is_moveing = true
  ├── 计算前方目标点（forwardTargetFromAngle）
  ├── 播放位移动画（线性补间）
  ├── 移动中碰撞检测：
  │   ├─ 碰撞其他实体 → stopAllActions → [COLLISION_STOP]
  │   └─ 无碰撞，到达边界 → [BOUNDARY_CHECK]
  └── 动画完成 → [BOUNDARY_CHECK]

[COLLISION_STOP] 碰撞停止
  ├── 吸附到最近网格中心
  ├── 更新网格占用
  ├── is_moveing = false
  └── → [CHECK]

[BOUNDARY_CHECK] 边界检测
  ├─ 动物位置超出网格范围（col<0 或 col≥12 或 row<0 或 row≥18）
  │   → winCount++ → 播放逃离动画 → 销毁节点 → [CHECK]
  └─ 未超出 → [COLLISION_STOP]

[CHECK] 状态检测 ★循环检测点★
  ├─ winCount ≥ winLength → [VICTORY]
  ├─ 倒计时 ≤ 0 → [FAIL]
  ├─ getMovableAnimals().length === 0 → [DEADLOCK]
  └─ 否则 → [IDLE]

[DEADLOCK] 死局处理
  ├── 显示提示文本
  ├─ 玩家使用道具 → 执行效果 → [IDLE]
  └─ 无道具可用 → [FAIL]

[VICTORY] 胜利结算
  ├── 停止倒计时
  ├── 播放胜利动画
  ├── 更新 localStorage（highest_score / recent_progress）
  └── 提供"下一关"按钮

[FAIL] 失败结算
  ├── 显示失败界面（当前进度百分比）
  ├─ 玩家选择复活 → 倒计时 +30秒 → [IDLE]
  └─ 玩家放弃 → 回到主页 / 重新开始
```

## 3. 系统模块

### 3.1 网格与移动模块

#### 模块目标
管理棋盘空间，处理动物的定向移动、碰撞检测和边界逃离判定。玩家通过点击触发动物移动，系统计算移动路径和碰撞结果。

#### 核心规则

**行为链：点击 → 校验 → 移动 → 碰撞/逃离**

| 步骤 | 触发条件 | 玩家可见行为 | 内部状态变化 | 持续时间 | 可否打断 | 下一步 | 失败回退 |
|------|---------|------------|------------|---------|---------|--------|---------|
| 1. 点击动物 | 玩家触摸动物节点 | 动物高亮（缩放至1.2倍后回弹） | 选中目标确定 | 即时 | — | 步骤2 | — |
| 2. 前方校验 | 步骤1完成 | 无可见变化 | 计算朝向目标点，检测前方是否有阻挡 | 即时 | — | 若可移动→步骤3；不可移动→取消高亮，回IDLE | 回IDLE |
| 3. 直线移动 | 校验通过 | 动物沿朝向直线滑动 | is_moveing=true，位置持续更新 | distance/speed ms | 不可打断（is_moveing锁） | 步骤4或步骤5 | — |
| 4. 碰撞停止 | 移动中碰到其他实体 | 动物停在碰撞点，粒子特效，轻微抖动 | stopAllActions，吸附网格中心，更新占用，is_moveing=false | 200ms反馈动画 | — | [CHECK] | — |
| 5. 边界逃离 | 动物移出网格范围 | 动物滑出画面，逃离轨迹特效 | winCount++，节点销毁，释放网格占用 | 400ms逃离动画 | — | [CHECK] | — |

**网格坐标系**：
- 原点：左上角
- col 向右递增（0 ~ 11），row 向下递增（0 ~ 17）
- 每只动物占 2×2 格，锚点在左上格

**朝向映射**：
- 0° = 上（row 递减方向）
- 90° = 右（col 递增方向）
- 180° = 下（row 递增方向）
- 270° = 左（col 递减方向）

**移动路径计算（forwardTargetFromAngle）**：
1. 根据动物当前 (col, row) 和 face 角度，计算射线方向
2. 沿射线检测最近的占用格或边界
3. 若最近障碍在网格内 → 目标点为障碍前一格（碰撞停止）
4. 若射线先到达边界 → 目标点为边界外（逃离消除）

**可移动性检测（getMovableAnimals）**：
- 遍历所有存活动物
- 对每只动物计算其朝向下一格是否被占用
- 若下一格为空或为边界外 → 可移动
- 返回所有可移动动物的列表

| 点击动物 | 缩放1.2倍回弹 | 点击音效 | 轻触震动 |
| 碰撞停止 | 粒子喷射 + 抖动 | 碰撞音效 | — |
| 边界逃离 | 白色轨迹拖尾 + 节点淡出 | 逃离音效 | — |
| 死局 | 提示文本弹出 | 提示音效 | — |

#### 显式否定与原型边界

**不做清单（游戏行为层面）**：
- ❌ 拖拽移动动物
- ❌ 动物碰撞后推动其他动物
- ❌ 消除后下落填充
- ❌ 多点触控同时移动多只动物
- ❌ 动物自动移动（所有移动必须由玩家点击触发）
- ❌ 对角线移动（只有上下左右四方向）

**原型简化范围**：
- 特殊动物（炸弹猪/时钟猪/睡羊/鸭子）原型阶段不实现，仅实现基础猪
- 关卡数量：原型实现前 5 关硬编码配置
- 道具次数：原型阶段每种道具默认 3 次/关，不消耗真实资源

#### 视觉语义

| 视觉特征 | 语义含义 | 验证方法 |
|---------|---------|---------|
| 动物上的三角形箭头方向 | 该动物的移动朝向 | 点击后观察移动方向应与箭头一致 |
| 黄绿交替棋盘格 | 网格区域，可放置动物 | 动物严格对齐格线 |
| 红色方块 | 不可移动障碍物 | 点击无反应，碰撞使移动动物停止 |
| 金色圆形按钮 | 可用道具 | 点击进入对应道具模式 |
| 灰色圆形按钮 | 不可用道具（次数耗尽） | 点击无反应 |
| 顶部数字 "N/M" | 已消除数/目标数 | 每次消除左边数字+1 |

#### 布局类型判定

本游戏为**固定网格棋盘布局**：
- 棋盘位置固定，居中显示
- 网格尺寸由关卡配置决定（标准12×18 / 教学8×8）
- 所有实体严格对齐网格
- 非自由拖拽布局，非物理堆叠布局

#### 关卡生成约束

**可解性保证**：
1. 关卡配置必须确保初始状态下 `getMovableAnimals().length > 0`（至少一只动物可移动）
2. 关卡应存在至少一个解法序列能在时间内清空所有目标动物（关卡设计阶段验证）

**生成规则**：
1. 所有动物占 2×2 格，不允许重叠放置
2. 动物不可放置在网格外
3. 障碍物占 1×1 格，不可与动物重叠
4. 每只动物必须配置 face 值（0/90/180/270）
5. winLength = 关卡中需要消除的动物总数

**死锁预防**：
- 运行时调用 `getMovableAnimals()`，若返回空数组且仍有未消除动物 → 触发死局流程
- 道具系统（洗牌重置朝向）作为死局恢复手段

### 3.2 胜负判定模块

#### 模块目标
跟踪关卡进度，判定胜利/失败条件，管理倒计时和死局检测。

#### 核心规则

| 步骤 | 触发条件 | 玩家可见行为 | 内部状态变化 | 持续时间 | 可否打断 | 下一步 | 失败回退 |
|------|---------|------------|------------|---------|---------|--------|---------|
| 1. 进度更新 | 动物冲出边界 | 顶部进度数字+1 | winCount++ | 即时 | — | 步骤2 | — |
| 2. 胜利检测 | winCount变化 | 若达标→胜利弹窗 | winCount≥winLength→停止计时 | 即时 | — | 胜利结算或继续 | — |
| 3. 时间流逝 | 每秒触发 | 倒计时数字-1，<10s变红 | pigClock-- | 1000ms/次 | 胜利时停止 | 步骤4 | — |
| 4. 超时检测 | pigClock≤0 | 失败界面弹出 | 游戏暂停 | 即时 | — | 失败结算 | — |
| 5. 死局检测 | 每次移动结束后 | 若死局→提示弹出 | getMovableAnimals检测 | 即时 | — | 死局处理或继续 | — |

**胜利条件**：`winCount >= winLength`
**失败条件 A**：`pigClock <= 0`（时间耗尽）
**失败条件 B**：`getMovableAnimals().length === 0` 且所有道具次数为 0（死局无道具）

**复活机制**：失败后提供一次复活机会，倒计时增加 30 秒，从当前状态继续。

#### 界面与交互

顶部状态栏显示：
- 左侧：暂停按钮（圆形，60×60px）
- 中央：关卡标识文本（"第 N 关"）
- 右侧上：进度文本（"winCount/winLength"，字号28px，白色 #FFFFFF）
- 右侧下：倒计时文本（"MM:SS"，字号32px，正常白色 #FFFFFF，<10s 红色 #FF4444）

#### 实现约束

- 倒计时使用 setInterval 每 1000ms 递减
- 胜利后必须立即停止计时器（clearInterval）
- 进度百分比 = `Math.floor(winCount / winLength * 100)`，失败界面显示

#### 关键反馈

| 事件 | 视觉 | 音效 |
|------|------|------|
| 胜利 | 弹窗从0.2缩放至1.1再至1.0（400ms，easeOutBack） | 胜利音效 |
| 失败 | 失败弹窗 + 当前进度百分比 | 失败音效 |
| 倒计时<10s | 数字变红 #FF4444 + 轻微脉冲缩放 | 滴答警告音（建议预留） |
| 复活 | 倒计时数字闪烁恢复 | 复活音效（建议预留） |

#### 显式否定与原型边界

- ❌ 不实现星级评价（无1-3星系统）
- ❌ 不实现步数限制（只有时间限制）
- ❌ 不实现连击加分（消除即计数，无连击倍率）

#### 视觉语义

- 进度文本颜色不变，仅数字变化
- 倒计时 <10s 时文本颜色从白变红，提示紧迫感

#### 布局类型判定

顶部状态栏为固定水平条，不参与游戏交互。

#### 关卡生成约束

- winLength 由关卡配置决定，必须等于该关可消除动物总数
- pigClock 初始值建议 90 秒（代码未明确硬编码，原型按此实现，后续调优）

### 3.3 道具模块

#### 模块目标
为玩家提供破局手段，缓解死局和时间压力。三种道具：移除（直接消除一只动物）、洗牌（随机重置所有动物朝向）、翻转（将一只动物顺时针旋转90°）。

#### 核心规则

| 步骤 | 触发条件 | 玩家可见行为 | 内部状态变化 | 持续时间 | 可否打断 | 下一步 | 失败回退 |
|------|---------|------------|------------|---------|---------|--------|---------|
| 1. 点击道具按钮 | 玩家点击底部道具按钮 | 按钮高亮，进入道具模式 | 设置对应标记（is_removeAnimal/is_flipAnimal） | 即时 | 点击空白取消 | 步骤2 | 回IDLE |
| 2a. 移除-点击目标 | 道具模式下点击动物 | 动物缩放至0后消失 | winCount++，节点销毁，使用计数+1 | 300ms | 不可打断 | 步骤3 | — |
| 2b. 洗牌-立即生效 | 点击洗牌按钮 | 所有动物旋转跳跃动画 | 随机重置所有face值，重算网格占用 | 500ms | 不可打断 | 步骤3 | — |
| 2c. 翻转-点击目标 | 道具模式下点击动物 | 动物旋转90°动画 | face=(face+90)%360 | 300ms | 不可打断 | 步骤3 | — |
| 3. 重置道具状态 | 效果执行完毕 | 按钮恢复常态 | 清除道具标记，使用计数检查 | 即时 | — | 回IDLE | — |

**使用限制**：
- 移除和翻转道具：每次激活最多使用 2 次，超过则自动退出道具模式（is_removeCount/is_flipCount ≥ 2 → 重置）
- 洗牌：点击即生效，无需选择目标
- 道具模式与移动模式互斥：道具激活时点击动物执行道具效果而非移动
- 原型阶段每种道具默认 3 次/关

#### 界面与交互

底部道具栏（3 个等距排列的圆形按钮）：

```
+--------------------------------------------------+
|      [ ✕ ]         [ ↻ ]         [ ↔ ]           |
|      移除            洗牌          翻转             |
|     剩余:3          剩余:3        剩余:3           |
+--------------------------------------------------+
```

- 按钮直径：80px
- 按钮间距：等分底部栏宽度
- 按钮底色：金色 #FFD700
- 图标色：白色 #FFFFFF
- 不可用状态：灰色 #999999
- 剩余次数：按钮下方文本，字号18px

#### 实现约束

- 道具按钮中心 Y 坐标：1234px（距底部 100px）
- 三按钮 X 坐标：187px / 375px / 563px（等分750px）
- 道具模式激活时，棋盘上动物显示可选中光标提示（移除模式：红色描边；翻转模式：蓝色描边）
- 洗牌动画期间设置 is_moveing=true 防止操作

#### 关键反馈

| 事件 | 视觉 | 时长 |
|------|------|------|
| 道具激活 | 按钮放大至1.1倍 + 金色光晕 | 持续至取消 |
| 移除效果 | 动物缩放至0 + 白色粒子爆散 | 300ms |
| 洗牌效果 | 所有动物上下跳跃 + 朝向箭头旋转 | 500ms |
| 翻转效果 | 目标动物旋转90° | 300ms |

#### 显式否定与原型边界

- ❌ 不支持批量移除（一次仅一个目标）
- ❌ 不支持连续翻转（每次使用仅翻转一只）
- ❌ 不支持撤销道具效果

#### 视觉语义

- 激活中的道具按钮有持续光晕，与未激活按钮形成对比
- 死局时道具按钮自动高亮闪烁提示

#### 布局类型判定

底部固定工具栏，不随棋盘滚动。

#### 关卡生成约束

道具次数为关卡固定配置，不依赖关卡生成算法。
