## 0.5 Coding Agent 导读

### A. 核心机制摘要

🎯 一句话核心：点击屏幕控制角色在垂直平台列间跳跃攀爬，配合翻牌记忆、找物配对、钓鱼等休闲小游戏获取资源。

⚡ 与常见模式的关键差异：
1. 不是三消/消除游戏，而是平台跳跃跑酷+小游戏合集（无网格、无cascade、无卡槽）
2. 不是物理引擎驱动跳跃，而是Tween补间动画精确控制位移（0.3秒）
3. 不是横向无尽跑酷，而是垂直攀爬（3-4列平台纵向排列）
4. 不是自由移动角色，而是点击即跳到下一列平台（X轴由平台决定）
5. 家园玩法不是自动养成，而是拖拽角色到家具区域触发互动

### B. 前置否定清单

🚫 以下游戏行为绝对不要实现：
❌ 网格交换/三消匹配——玩家不应看到任何网格消除行为
❌ 底部卡槽收集——玩家不应看到横排卡槽区
❌ cascade下落补位——消失后不补位，空位保留
❌ 物理碰撞弹射——角色使用Tween位移，不是物理模拟
❌ 自动寻路AI移动——角色仅响应玩家输入
❌ 横向无尽卷轴——主玩法为纵向攀爬
❌ 离线挂机结算——容器倒计时仅在线计算
⚠️ 包含以上任何一项，说明核心设计理解错误。

### C. 反例描述

🚨 错误实现特征：

1. ❌ 错误现象：角色在横向轨道上自动跑动，玩家点击跳跃躲避障碍
   ✅ 正确现象：角色在纵向3-4列平台间向上跳跃，X轴由目标平台决定
   📌 验证方法：启动主玩法关卡 → 观察角色是否只做垂直方向位移（Y轴递增），且水平位移仅切换到下一平台列

2. ❌ 错误现象：点击屏幕后角色持续移动或有惯性
   ✅ 正确现象：单次点击触发一次跳跃Tween（0.3秒），结束后角色静止等待下次输入
   📌 验证方法：点击一次 → 观察角色跳到下一平台后完全静止 → Tween进行中再次点击无响应

3. ❌ 错误现象：翻牌小游戏中卡片飞向底部卡槽进行三张匹配消除
   ✅ 正确现象：翻开两张相同卡片后原地消失，无卡槽收集
   📌 验证方法：翻开两张相同卡片 → 卡片在原位淡出消失 → 底部无任何收集槽

4. ❌ 错误现象：角色跳跃后使用物理引擎计算抛物线和碰撞
   ✅ 正确现象：角色通过Tween线性/缓动插值直接移动到目标坐标
   📌 验证方法：观察角色跳跃轨迹 → 应为平滑补间动画，非自由落体抛物线

5. ❌ 错误现象：平台生成为完全随机散布
   ✅ 正确现象：平台严格按3列或4列垂直排列，列间距均匀
   📌 验证方法：启动关卡1-3 → 平台应为4列等距排列 → 关卡4+ → 切换为3列等距排列

6. ❌ 错误现象：家园中角色自动走向家具
   ✅ 正确现象：玩家必须手动拖拽角色到家具区域，释放后检测isInArea
   📌 验证方法：不拖拽角色 → 角色永远不会自动移动到家具处

## 1. 游戏概述

### 1.1 一句话核心玩法

玩家在垂直平台列间通过点击跳跃攀爬突破高度记录，同时在横版家园中拖拽角色与家具互动，并游玩翻牌记忆、找物配对、钓鱼等休闲小游戏获取金币与资源。

### 1.2 游戏类型

- **主类型**: 2D平台跳跃跑酷（Platformer Runner）
- **子类型**: 休闲小游戏合集（Mini-Game Collection）
- **辅助类型**: 轻养成/模拟经营（横版家园）

### 1.3 核心动词

点击、跳跃、攀爬、拖拽、翻牌、配对、捕捉

### 1.4 体验关键词

轻松上手、垂直攀爬挑战、多样玩法切换、萌系治愈、碎片时间友好

### 1.5 交互模式

- 主玩法：单点触控（点击屏幕触发跳跃）
- 家园：拖拽（触摸移动角色到目标区域）
- 翻牌：单点触控（点击翻开卡片）
- 找物：单点触控（点击目标物品）
- 钓鱼：单点触控（点击发射钩子）

### 1.6 参数刚性标记

| 参数 | 值 | 刚性 |
|------|---|------|
| 跳跃Tween时长 | 0.3秒 | 🔒核心 |
| 早期关卡列数 | 4列 | 🔒核心 |
| 后期关卡列数 | 3列 | 🔒核心 |
| platformSpacingY | 130px | 🔧偏好 |
| 钓鱼限时 | 50秒 | 🔧偏好 |
| 翻牌网格 | 4×4 | 🔧偏好 |
| 家园轨道数 | 7条 | 💡建议 |

## 2. 核心循环

### 2.1 主玩法循环（平台跳跃）

```
[等待输入] → 玩家点击屏幕
    ↓
[跳跃执行] → Tween动画0.3秒位移至目标平台坐标
    ↓
[落地检测] → 角色坐标是否在平台碰撞区内？
    ↓ 是                          ↓ 否
[攀爬成功]                     [游戏结束]
    ↓                              ↓
[平台层下移] → 相机跟随        更新highest_score
    ↓                          显示结束界面
[生成新平台] → layerNum++
    ↓
[检测关卡完成？]
    ↓ 是           ↓ 否
[关卡通关]      回到[等待输入] ★循环
```

### 2.2 子玩法循环（翻牌记忆）

```
[初始化] → 生成4×4网格，16张卡片（8对）背面朝上
    ↓
[等待输入] → 玩家点击卡片A → 翻转动画200ms → 显示正面
    ↓
[等待第二张] → 玩家点击卡片B → 翻转动画200ms → 显示正面
    ↓
[配对判定] → A与B图案相同？
    ↓ 是                    ↓ 否
[配对成功]               [配对失败]
两张淡出消失300ms         延迟800ms后翻回背面200ms
    ↓                        ↓
[全部完成？]             回到[等待输入] ★循环
    ↓ 是
[通关结算] → 显示得分
```

### 2.3 子玩法循环（找物配对）

```
[初始化] → 场景散布8-12个物品，底部显示3个目标
    ↓
[等待输入] → 玩家点击场景中物品
    ↓
[匹配判定] → 点击物品是否为目标之一？
    ↓ 是                    ↓ 否
[找到目标]               无反应/轻微抖动
物品高亮后淡出300ms
目标计数-1
    ↓
[全部找到？]
    ↓ 是
[通关结算]
```

### 2.4 子玩法循环（钓鱼）

```
[初始化] → 水域生成鱼类4-6条+障碍2-4个，倒计时50秒启动
    ↓
[等待输入] → 玩家点击屏幕 → 钩子向下延伸
    ↓
[碰撞检测] → 钩子触碰目标？
    ↓ 鱼类              ↓ 炸弹/鲨鱼         ↓ 未触碰到底
[捕获成功]           [游戏结束]           [钩子收回]
分数+1，钩子收回      显示结算             回到[等待输入]
    ↓
[倒计时归零？]
    ↓ 是
[时间结束] → 显示最终得分
```

### 2.5 核心张力

| 玩法 | 张力来源 | 描述 |
|------|---------|------|
| 平台跳跃 | 精准度 vs 递增难度 | 随高度增加平台变少（4列→3列）、出现破碎平台，一次失误即结束 |
| 翻牌记忆 | 记忆力 vs 卡片数量 | 需记住16张卡片位置并高效配对 |
| 找物配对 | 观察力 vs 干扰物 | 在多个相似物品中快速辨别目标 |
| 钓鱼 | 操作时机 vs 障碍物 | 在运动障碍物间找到安全窗口捕捉鱼类 |

## 3. 系统模块

### 3.1 平台跳跃模块（主玩法）

#### 模块目标

玩家通过精准点击控制角色在垂直平台列间跳跃攀爬，突破历史高度记录并收集金币。

#### 核心规则

| 步骤 | 触发条件 | 输入对象 | 输出结果 | 动画/反馈 | 耗时 | 失败条件 | 状态变化 |
|------|---------|---------|---------|----------|------|---------|---------|
| 1. 玩家点击 | 屏幕任意位置被触摸且当前非跳跃中 | 触摸事件坐标 | 计算目标平台坐标 | 无 | 0ms | 无 | READY→JUMPING |
| 2. 跳跃执行 | 目标坐标确定 | 角色节点 | 角色Tween位移到目标(x,y) | Tween缓动动画 | 300ms | 无 | JUMPING中 |
| 3. 落地检测 | Tween动画结束 | 角色终点坐标 | 判定是否在平台碰撞区内 | 无 | 0ms | 未踩中平台 | JUMPING→LANDED/GAME_OVER |
| 4. 平台层下移 | 落地成功 | 平台层节点 | 整体下移platformSpacingY | 平滑下移动画200ms | 200ms | 无 | LANDED→PLATFORM_SHIFT |
| 5. 生成新平台 | 平台下移完成 | levelNum | 在顶部生成新平台行 | 新平台淡入100ms | 100ms | 无 | PLATFORM_SHIFT→READY |
| 6. 关卡递增 | layerNum达到目标 | layerNum | levelNum++ | 关卡完成提示 | - | 无 | →LEVEL_COMPLETE |

**平台选择逻辑**：
- 跳跃目标为当前角色正上方最近的一个平台
- X轴移动到该平台的中心X坐标
- Y轴移动距离等于platformSpacingY（130px）

**平台类型分布**：
- 普通平台：稳定不变，关卡1-3占比100%
- 破碎平台：角色踩上后1秒内倒塌消失，关卡4+出现，占比20%递增至40%
- 移动平台（原型中不实现，标记为延期）

#### 界面与交互

```
┌─────────────────────────────────┐ ← 顶部状态栏 10%
│  [||]  ★★★☆☆     💰 128       │
├─────────────────────────────────┤
│ ┃                               │ ← 左侧进度条 3%宽
│ ┃    [平台]    [平台]   [平台]  │
│ ┃                               │
│ ┃         [平台]    [平台]      │ ← 主操作区 75%高
│ ┃                               │
│ ┃    [平台]  [角色]  [平台]     │
│ ┃              ↑                │
│ ┃         [平台]    [平台]      │
│ ┃                               │
├─────────────────────────────────┤
│  (←)                    (→)     │ ← 底部控制区 15%
└─────────────────────────────────┘
```

**交互状态表**：

| 状态 | 可交互 | 输入响应 | 视觉提示 |
|------|--------|---------|---------|
| READY | 是 | 点击→触发跳跃 | 角色待机呼吸动画 |
| JUMPING | 否 | 忽略所有输入 | 角色跳跃姿态 |
| LANDED | 短暂（自动转换） | 无 | 落地微弹 |
| GAME_OVER | 是 | 点击重试按钮 | 结束面板覆盖 |

#### 实现约束

**空间分区（百分比）**：
- 顶部状态栏：0%-10% Y轴
- 左侧进度条：0%-3.3% X轴（25px/750px）
- 主操作区：10%-85% Y轴，3.3%-100% X轴
- 底部控制区：85%-100% Y轴

**交互判定**：
- 落地碰撞检测：角色中心点与平台矩形区域的AABB碰撞，平台碰撞区为60×30px（比视觉略大10px容差）
- 输入锁定：Tween执行期间（300ms）忽略所有触摸事件
- 金币拾取：角色中心与金币中心距离<40px时自动拾取

**动画参数**：
| 动画名 | 触发时机 | 时长 | 缓动 | 说明 |
|--------|---------|------|------|------|
| jump_tween | 点击屏幕 | 300ms | easeOut | 角色位移到目标平台 |
| platform_shift | 落地成功 | 200ms | easeInOut | 平台层整体下移 |
| platform_fade_in | 新平台生成 | 100ms | linear | 新平台从透明到不透明 |
| crumble | 踩上破碎平台1s后 | 500ms | linear | 平台碎裂消失 |
| coin_collect | 角色接近金币 | 200ms | easeIn | 金币飞向顶部计数器 |
| game_over_fall | 未踩中平台 | 600ms | easeIn | 角色向下掉落消失 |

| 事件 | 视觉反馈 | 音效 | 震动 |

| 跳跃触发 | 角色切换跳跃姿态 | 轻快弹跳声 | 无 |
| 落地成功 | 平台微压缩10%后回弹 | 轻"tap"声 | 无 |

#### 显式否定与原型边界

**不做清单**：
- ❌ 不实现物理引擎跳跃（使用Tween精确控制）
- ❌ 不实现左右移动控制（水平位置由目标平台决定）
- ❌ 不实现无限复活（游戏结束需重新开始或消耗本地积累的金币复活，最多1次/局）
- ❌ 不实现道具系统（原型阶段无加速/护盾/磁铁等）
- ❌ 不实现移动平台（标记[延期]）

**原型简化范围**：
- 角色外观使用纯代码绘制（圆形头+矩形身体+三角耳朵），不依赖Spine
- 平台使用圆角矩形+简单装饰绘制
- 破碎平台通过描边裂纹线条区分

#### 视觉语义

**布局验证特征**：
- 平台必须严格对齐到列网格（4列或3列等分屏幕宽度）
- 角色始终位于某一平台正上方（跳跃中除外）
- 进度条必须在最左侧且不遮挡操作区
- 底部方向键仅为视觉装饰（实际输入为全屏点击）

**层级关系**：
背景(z=0) → 进度条(z=5) → 平台(z=10) → 金币(z=15) → 角色(z=20) → UI按钮(z=50) → 弹窗(z=100)

#### 布局类型判定

本模块为**纵向固定列布局**：
- 列数固定（3或4），X轴位置预计算
- Y轴等间距排列，间距platformSpacingY=130px
- 相机跟随角色Y轴，平台动态生成/回收
- 区别于"自由散布"和"网格填充"布局

#### 关卡生成约束

**生成算法**：
```
输入: levelNum
输出: 平台布局配置

1. columns = (levelNum <= 3) ? 4 : 3
2. 计算列X坐标: colX[i] = screenWidth/(columns+1) * (i+1)
3. 每层随机选择1-2个列放置平台
4. 平台类型概率:
   - 普通 = max(60%, 100% - levelNum*5%)
   - 破碎 = min(40%, levelNum*5%)
5. 确保相邻两层至少有一个平台列相邻或相同（可达性保证）
6. 每3-5层放置一个金币（随机列）
```

**可达性保证**：
- 相邻两层平台的列位置差不超过1列（角色能跳到）
- 不允许出现连续3层只有破碎平台的情况
- 每关至少20层平台，目标层数 = 20 + levelNum * 5

| 捕获鱼 | 鱼变金色跟随上移+"+1"飘字 | 成功捕获音 | 无 |

#### 显式否定与原型边界

- ❌ 不实现钩子方向控制（固定垂直向下）
- ❌ 不实现多条钩子同时发射
- ❌ 不实现鱼类种类差异化得分（原型统一+1分）
- ❌ 不实现道具（加速/护盾）

#### 视觉语义

- 水面线：波浪形sin曲线，浅蓝#87CEEB，振幅5px，动画循环
- 水域背景：渐变从#87CEEB(顶)到#2C5F7C(底)
- 气泡装饰：随机上升的圆形气泡3-5个，白色半透明，直径5-10px
- 鱼类：橙红#FF6348椭圆体+三角尾巴
- 炸弹：黑色#2D3436圆形+白色眼睛+顶部引线
- 鲨鱼：紫色#6C5CE7流线体+三角背鳍+锯齿嘴

#### 布局类型判定

本模块为**自由运动场景**：生物在矩形水域内动态移动，无网格约束。

#### 关卡生成约束

```
1. 鱼类数量: 4-6条，Y轴均匀分布在水域区域
2. 炸弹数量: 1-3个，不与鱼初始位置重叠（最小间距60px）
3. 鲨鱼数量: 0-2条，在较深水域层（Y轴下半区）
4. 所有生物初始位置检查：互不重叠
5. 确保至少存在1条可安全到达的鱼（垂直路径无障碍遮挡）
```

## 4. 资源与经济

### 4.1 资源表

| 资源名 | 获取来源 | 消耗去向 | 存储方式 |
|--------|---------|---------|---------|
| 金币 | 平台跳跃收集、小游戏通关奖励、容器收获 | 家园家具解锁（延期）、角色装扮（延期） | recent_progress.coins |
| 最高分 | 平台跳跃结算 | 无消耗（记录用） | highest_score |
| 关卡进度 | 平台跳跃通关 | 解锁下一关 | recent_progress.level |
| 情绪值 | 家具互动 | 自然衰减 | 运行时内存（不持久化） |

### 4.2 经济循环

```
[平台跳跃] → 金币(3-5/局) + 最高分记录
      ↓
[小游戏] → 金币(5-10/局)
      ↓
[家园容器] → 金币(按等级1-10/次)
      ↓
[消耗出口] → 原型阶段无消耗出口，金币仅累积展示
```

**原型简化说明**：原型中金币仅作为成就记录，无实际消耗系统。后续版本可添加装扮购买作为金币消耗出口。

## 7. 硬性边界合规声明

本文档仅包含核心玩法与本地进度，不包含红线外系统。

所有数据存储限于 localStorage 两个 key：
- `highest_score`：平台跳跃历史最高分（数字）
- `recent_progress`：关卡进度、金币累计、解锁状态（JSON对象）

无任何网络请求、无任何第三方SDK依赖、无任何用户间交互功能。
