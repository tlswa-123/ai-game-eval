## 0.5 Coding Agent 导读

### A. 核心机制摘要

```
🎯 一句话核心：拖拽交换相邻彩色方块形成3+同色连线→消除→重力下落→
   新方块填充→连锁检测循环(cascade)，在倒计时/移动墙压力下收集掉落物达标

⚡ 与其他消除类的关键差异：
1. 不是点击即消而是**拖拽交换匹配**——必须拖拽交换两个相邻方块形成3+连线
2. 不是消完等操作而是**连锁cascade**——下落后自动检测新匹配→继续消除→循环
3. 不是无限操作而是**双压力驱动**——倒计时时间压力 + 移动墙空间压力（关卡选配）
4. 不是交换即消耗而是**失败回退**——未形成3连的交换必须动画回退
5. 不是固定棋盘而是**可变网格尺寸**——网格随关卡变化（11×10/9×10/5×20/13×9等）
6. 不是随机初始而是**无初始匹配**——生成后检测→有3连→重新随机该位置
7. 首期**无特殊道具生成**——代码证据不足，仅实现基础三连消除
```

### B. 前置否定清单

```
🚫 以下游戏行为绝对不要实现：
❌ 点击直接消除——必须拖拽交换相邻方块
❌ 斜向交换/斜向消除——仅支持上下左右四方向
❌ 底部收集槽/卡槽区——无羊了个羊式底部7格收集槽，消除直接触发重力填充
❌ 物理碰撞触发消除——物理仅用于掉落物（球体）触底计分，非消除机制
❌ 特殊方块技能（四消炸弹/彩虹方块）——代码证据不足，首期不做
❌ 交换失败不回退——必须动画回退
❌ 消除后留空不填补——消除后必须重力下落+顶部生成新方块
❌ 下落后不检测连锁——cascade是核心循环
⚠️ 包含以上任何一项，说明核心设计理解错误。
```

### C. 反例描述

```
🚨 错误实现特征：

1. ❌ 错误现象：实现"点击三个相同颜色方块消除"（羊了个羊式）
   ✅ 正确现象：必须拖拽交换相邻方块，仅当交换后形成三连才消除
   📌 验证方法：点击单个方块→不应直接消除→拖拽到相邻位置→形成3连→消除

2. ❌ 错误现象：消除后空位永久保留，不填补（F02a式孔隙永存）
   ✅ 正确现象：消除后上方方块重力下落填补→顶部生成新方块
   📌 验证方法：消除一组方块→上方方块应自动下落→顶部应出现新方块

3. ❌ 错误现象：下落后不检测新匹配，直接回到待机
   ✅ 正确现象：下落完成→全盘扫描→有匹配→继续消除(cascade)
   📌 验证方法：消除后下落形成新3连→应自动触发二次消除

4. ❌ 错误现象：支持斜向交换或跳格交换
   ✅ 正确现象：仅允许上下左右四方向与直接相邻方块交换
   📌 验证方法：尝试斜向拖拽→应无响应或回弹

5. ❌ 错误现象：物理碰撞触发方块消除
   ✅ 正确现象：物理碰撞仅用于球体触底计分，消除由jiancesanlian匹配检测触发
   📌 验证方法：球体下落触底→仅增加分数→不触发棋盘消除

6. ❌ 错误现象：初始棋盘已有3连匹配
   ✅ 正确现象：生成后检测→有3连→重新随机该位置颜色
   📌 验证方法：开局→棋盘上不应有任何已存在的3连组

7. ❌ 错误现象：消除/下落动画未完成就允许新操作
   ✅ 正确现象：动画播放期间锁定输入(cantouch=false)，完成后才解锁
   📌 验证方法：消除动画播放中快速拖拽→不应响应新交换
```

## 3. 系统模块

### 3.1 输入与交换系统

#### 模块目标
玩家通过拖拽手势交换相邻方块，发起一次消除尝试。

#### 核心规则

**Step1: 触摸选中**

| 维度 | 描述 |
|------|------|
| **触发** | 手指触摸屏幕(onTouchStart) |
| **前置检查** | ✅ cantouch=true且IsGameOver=false ✅ 触摸点在方块多边形内(pointInPolygon) ❌ 动画播放中→不响应 |
| **执行** | 记录curr_nd(当前选中节点)+startClickPos(起始位置) |
| **后果** | 方块放大至110%+白色描边3px高亮 |
| **终止** | 选中确认 |
| **显式否定** | ❌不支持长按多选 ❌不支持空白区域触发 |
| **时序** | 即时响应(<16ms) |

**Step2: 拖拽判定**

| 维度 | 描述 |
|------|------|
| **触发** | 手指移动(onTouchMove) |
| **前置检查** | ✅ 已选中方块 ✅ cantouch=true |
| **执行** | 计算offset=当前位置-起始位置，确定拖拽方向(上/下/左/右)，计算arr_neibor四方向邻居 |
| **后果** | 方块跟随手指移动(限制在单格范围内item_wid) |
| **终止** | 手指释放(onTouchEnd) |
| **显式否定** | ❌不支持斜向拖拽 ❌拖拽范围不超过单格 |
| **时序** | 逐帧跟随 |

**Step3: 交换/回弹**

| 维度 | 描述 |
|------|------|
| **触发** | 手指释放(onTouchEnd) |
| **前置检查** | ✅ Math.abs(offset) > 0.5 * item_wid(超过半格) ✅ 目标方向有有效邻居(非空/非边界) |
| **执行-交换** | swap_ndpos交换geziData中两个位置数值→Tween位移动画0.2s→调用jiancesanlian检测 |
| **后果-成功** | 形成3+连线→进入消除流程 |
| **后果-失败** | 未形成3连→Tween回弹动画0.2s→回到原位→不消耗任何资源 |
| **终止** | 交换/回弹动画完成 |
| **显式否定** | ❌不支持对角交换 ❌不支持跳格交换 ❌动画期间不响应新输入 |
| **时序** | 交换动画200ms(ease-in-out)/回弹动画200ms(ease-out) |

- 顶部状态栏：0%-8%H（0-107px）
- 棋盘区域：12%-72%H（160-960px），水平居中90%W（37-712px）
- 底部功能区：80%-95%H（1067-1267px）

**交互判定**：

- 交换阈值：拖拽偏移量 > 0.5 × item_wid
- 方向判定：取水平/垂直偏移绝对值较大者为拖拽方向
- 邻居数组：arr_neibor=[上,下,左,右]，仅4方向

**动画参数**：
- 交换动画：200ms，ease-in-out缓动
- 回弹动画：200ms，ease-out缓动
- 选中放大：100ms，scale 1.0→1.1

### 3.2 匹配与消除系统

#### 模块目标
检测3+同色连线并执行消除，触发连锁循环(cascade)，产生掉落物球体用于计分。

#### 核心规则

**Step1: 匹配检测(jiancesanlian)**

| 维度 | 描述 |
|------|------|
| **触发** | 交换动画完成后 / 下落填充完成后(cascade) |
| **前置检查** | ✅ 交换/下落动画已完成 ✅ 未达最大连锁次数(20) |
| **执行** | 横向扫描：遍历每行，检查连续同色≥3；纵向扫描：遍历每列，检查连续同色≥3 |
| **后果-有匹配** | 标记匹配方块geziData[row][col]=[-1,-1]，收集至deleteNds数组 |
| **后果-无匹配** | 若为交换后首次检测→回弹；若为cascade→结束连锁回到IDLE |
| **终止** | 扫描完成 |
| **显式否定** | ❌不检测斜向连线 ❌不检测L/T形特殊匹配(首期) |
| **时序** | 即时计算(<16ms) |

**Step2: 消除动画**

| 维度 | 描述 |
|------|------|
| **触发** | 匹配标记完成后 |
| **前置检查** | ✅ deleteNds数组非空 |
| **执行** | 被标记方块播放缩放消失动画(scale→0)+粒子爆散→节点回收至对象池 |
| **后果** | 棋盘出现空位，触发掉落物生成(条件)+重力下落 |
| **终止** | 所有消除动画完成 |
| **显式否定** | ❌消除期间不允许新输入(cantouch=false) |
| **时序** | 200ms缩放消失+粒子生命周期500ms |

**Step3: 掉落物生成(条件触发)**

| 维度 | 描述 |
|------|------|
| **触发** | 消除的方块含掉落物标记(特定类型如"石头"/"芒果") |
| **前置检查** | ✅ 方块类型匹配掉落物规则 |
| **执行** | 在消除位置世界坐标生成yuan球体节点，赋予RigidBody组件，受重力物理下落 |
| **后果** | 球体触底(onBeginContact)→curr_score++→球体回收 |
| **终止** | 球体触底或超出屏幕 |
| **显式否定** | ❌球体不触发棋盘消除 ❌IsGameOver时球体直接回收不计分 |
| **时序** | 物理模拟，约500-1000ms触底 |

**Step4: 重力下落(fangkuaiDown)**

| 维度 | 描述 |
|------|------|
| **触发** | 消除动画完成后 |
| **前置检查** | ✅ 网格中存在空位(geziData=[-1,-1]) |
| **执行** | 按列独立处理：从底部向上遍历→空位上方非空方块下移填补→更新geziData索引 |
| **后果** | 空位被填补，列顶出现新空位 |
| **终止** | 列内无空位可填补 |
| **显式否定** | ❌不支持横向滑动填补 ❌不支持对角下落 |
| **时序** | 每格50ms下落(ease-in-quad)+落地回弹(bounce 100ms) |

**Step5: 顶部新块生成(getsjnum)**

| 维度 | 描述 |
|------|------|
| **触发** | 列内下落完成，顶部存在空位 |
| **前置检查** | ✅ 列顶有空位 |
| **执行** | 随机生成颜色(color_count种)→检查与下方和左方邻居颜色不同→从屏幕外上方掉入 |
| **后果** | 所有空位填满 |
| **终止** | 网格满 |
| **显式否定** | ❌新块颜色不得与下方/左方相邻形成初始3连 |
| **时序** | 入场动画200ms(bounce) |

**Step6: 连锁检测(cascade)**

| 维度 | 描述 |
|------|------|
| **触发** | 下落+填充动画全部完成 |
| **前置检查** | ✅ 下落动画完成 ✅ cascade次数<20 |
| **执行** | 全盘扫描(同Step1)→有新匹配→标记消除→下落→填充→再扫描→循环 |
| **后果** | 连锁N次，每次cascade分数递增 |
| **终止** | 无新匹配 或 cascade达到上限20次 |
| **显式否定** | ❌cascade期间不允许玩家操作 |
| **时序** | 每层cascade +200ms |

#### 界面与交互

消除系统无独立界面，作用于棋盘网格区域内。

#### 实现约束

**匹配算法**：
```
function jiancesanlian():
    matches = []
    // 横向扫描
    for row in 0..map_Hight-1:
        for col in 0..map_Width-3:
            color = geziData[row][col][0]
            if color != -1 and
               color == geziData[row][col+1][0] and
               color == geziData[row][col+2][0]:
                标记(row,col), (row,col+1), (row,col+2)
    // 纵向扫描
    for col in 0..map_Width-1:
        for row in 0..map_Hight-3:
            color = geziData[row][col][0]
            if color != -1 and
               color == geziData[row+1][col][0] and
               color == geziData[row+2][col][0]:
                标记(row,col), (row+1,col), (row+2,col)
    return 去重后matches
```

**下落算法**：
```
function fangkuaiDown():
    for col in 0..map_Width-1:
        emptyCount = 0
        for row from map_Hight-1 down to 0:
            if geziData[row][col] == [-1,-1]:
                emptyCount++
            else if emptyCount > 0:
                geziData[row+emptyCount][col] = geziData[row][col]
                geziData[row][col] = [-1,-1]
        for i in 0..emptyCount-1:
            geziData[i][col] = getsjnum(i, col)
```

**状态锁定**：
- isCanMove / cantouch：交换/消除/下落/cascade全程锁定
- IsGameOver：游戏结束后永久锁定
- isStartChange：动画播放锁
- isDownGrid：炸弹道具使用锁

## 6. 视觉与审美

### 6.1 整体风格

- **目标审美**：3D卡通圆润感的2D渲染风格——方块具有圆角立方体外观、黑色轮廓线勾勒、纯色哑光材质，整体明快活泼
- **风格关键词**：马卡龙、卡通圆润、高饱和、黑描边、哑光平涂
- **形状语言**：圆角为主（方块r=8px、按钮r=30px、弹窗r=16px），无尖锐棱角，传达亲和安全感
- **材质倾向**：纯色哑光填充+细微内光泽(顶部5%区域rgba(255,255,255,0.15))，无金属/玻璃/渐变材质
- **主题叙事**：猫咪主题场景化包装——顶部装饰有卡通猫爪元素，方块可设计为猫罐头/毛线球等图案（纯代码绘制用几何形状抽象表达）

## 7. 硬性边界合规声明

本文档仅包含核心玩法与本地进度，不包含红线外系统。

**本地持久化规范**：
- localStorage 仅允许保存两个 key：`highest_score` 与 `recent_progress`
- 无云端同步、无远程校验

**运行形态**：纯本地单机核心玩法体验。
